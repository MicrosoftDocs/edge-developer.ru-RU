# <a name="publish-your-extension"></a>Публикация расширения  

После разработки и тестирования расширения вы будете готовы к распространению расширения. Для распространения расширения используйте магазин надстройок Microsoft Edge.  Чтобы освободить существующее расширение Chromium для пользователей Microsoft Edge, перейдите в порт существующего расширения [Chromium.][PortChromiumExtension]  

Опубликуйте расширение в магазине надстройок Microsoft Edge, чтобы увеличить его охват и сделать его доступным для пользователей Microsoft Edge.  В этой статье предоставляется процесс отправки расширения в магазин надстройок Microsoft Edge.  

## <a name="before-you-begin"></a>Перед началом работы  

Рабочий прототип расширения должен быть готов.  Сведения о создании расширения можно найти в руководстве по началу [работы.][ExtensionsGettingStarted]  

Чтобы опубликовать расширение в магазине надстройок Microsoft Edge, используйте свою активную учетную запись разработчика в [Центре партнеров.][MicrosoftPartnerCenter]  Если у вас нет учетной записи разработчика, создайте новую учетную запись разработчика.  Чтобы открыть новую учетную запись разработчика и зарегистрироваться в программе надстройок Microsoft Edge, перейдите к [регистрации разработчика.][DeveloperRegistration]  

Создайте почтовый файл, который представляет пакет расширения.  Пакет расширения должен включать следующие файлы.  

*   Манифест расширения, в который указываются такие сведения, как имя расширения, краткое описание, разрешения и язык по умолчанию.  
*   Изображения и другие файлы, необходимые для расширения.  

Следующие поля манифеста автоматически включаются в сведения о листинге магазина.  Поля только для чтения на странице **Списки Магазина.**  Страница списков магазина описана позже в этой статье.  Убедитесь, что значения поля совпадают с предпочтительным отображением на странице сведений о магазине перед отправкой пакета в Центр партнеров.  В качестве примера кода, необходимого для файла манифеста, просмотрите основы файла манифеста.  

*   `Name` поле в файле манифеста, которое является именем **Display** на странице сведений магазина.  
*   `Description` поле в файле манифеста, которое является **кратким описанием** на странице сведения магазина.  Предокажи краткое броский описание, отображаемое в верхней части списка для расширения.  При включении краткое описание, указанное в файле манифеста расширения, отображается в списке магазина.  Если краткое описание не включено в файл манифеста, отображаются первые несколько строк Описания.  Укажи краткое описание, чтобы избежать повторения контента на странице списка магазина.  

## <a name="submit-your-extension-to-microsoft-edge-add-ons-store"></a>Отправка расширения в магазин надстройок Microsoft Edge  

Чтобы отправить расширение в [Центр партнеров,][MicrosoftPartnerCenter]используйте следующие действия.  

#### <a name="step-1--start-a-new-submission"></a>Шаг 1. Запуск новой отправки  

Перейдите к панели [мониторинга разработчиков][MicrosoftPartnerCenter] и **выберите Создание нового расширения** на странице **Обзор.**  

#### <a name="step-2--upload-the-extension-package"></a>Шаг 2. Отправка пакета расширения  

Используйте **страницу Пакеты,** чтобы загрузить почтовый файл пакета расширения.  Вы можете загружать только один пакет одновременно.  Вы не сможете продолжить отправку, если загрузка пакета не будет успешной на странице **Пакеты.**  

Чтобы загрузить пакет, выберите и перетащите пакет в поле отправки. Вы также можете выбрать **Просмотр файлов**.  После отправки пакета пакет проверяется.  После успешной проверки просмотрите сведения о расширении и выберите **Далее** для продолжения.  Если имеются ошибки проверки, устраняйте проблемы и попробуйте загрузить еще раз.  

#### <a name="step-3--provide-availability-details"></a>Шаг 3. Предоставление сведений о доступности  

На странице **Доступность введите** следующую информацию о доступности расширения.  

##### <a name="visibility"></a>Видимость  

Выберите один из следующих вариантов видимости, чтобы определить, можно ли обнаружить расширение в магазине надстройок Microsoft Edge.  

*   `Public` \(по умолчанию\)  
    Общедоступный доступ позволяет всем пользователям открывать расширение с помощью поиска, просмотра в магазине надстройок Microsoft Edge или использования URL-адреса списка в вашем расширении в магазине надстройок Microsoft Edge.  URL-адрес списка доступен на панели мониторинга Центра партнеров на странице Обзор **расширения.**  
    
*   `Hidden`  
    Hidden удаляет расширения из результатов поиска или просмотра в магазине надстройок Microsoft Edge.  Чтобы распространять скрытые расширения в магазине надстройок Microsoft Edge, необходимо поделиться URL-адресом списка с расширением с вашими клиентами.  
    
> [!NOTE]
> Вы можете изменить видимость расширения с Общедоступный на **Скрытый** . ****  Пользователи, которые установили расширение во время установки видимости для общего доступа к вашему расширению и получают все обновления, доступные на веб-сайте надстройок Microsoft Edge.  

##### <a name="markets"></a>Страны  

Определите конкретные рынки, на которых планируется предлагать расширение.  Параметр по умолчанию для рынков — это все рынки, которые включают все будущие рынки, которые будут добавлены позже.  Чтобы выбрать конкретные рынки, выберите **Рынки изменений.**  Чтобы исключить отдельные рынки, выберите **unselect и** добавьте отдельные рынки по вашему выбору.  

> [!NOTE]
> Вы можете изменить рынки, на которых предлагается расширение.  Пользователь, устанавливавший расширение, пока оно доступно на рынке пользователя, сохраняет доступ к вашему расширению.  Однако пользователь не имеет доступа к будущим обновлениям, отправленным в магазин надстройок Microsoft Edge.  

Выберите **Сохранить,** чтобы продолжить работу в разделе **Свойства.**  

#### <a name="step-4-select-properties-for-your-extension"></a>Шаг 4. Выбор свойств для расширения  

На странице **Свойства введите**следующую информацию, чтобы указать свойства расширения.  Свойства отображаются пользователям в магазине надстройок Microsoft Edge.  

| Имя свойства расширения | Описание |  
|:--- |:--- |  
| Категория \(required\) | Категория, которая лучше всего описывает расширение.  Включение расширения в правую категорию помогает пользователям легко найти ваше расширение и узнать больше об этом.  |  
| Требования политики конфиденциальности \(required\) | Указать, имеется ли доступ к расширению, собирает или передает какие-либо персональные данные.  Расширение может привести к сбой в этапе сертификации, если вы выберете **да,** и вы не предоставите `Privacy policy URL` .  |  
| URL-адрес с политикой конфиденциальности | Допустимый URL-адрес политики конфиденциальности, чтобы сообщить, как ваше расширение следует законам и правилам конфиденциальности.  Вы отвечаете за то, чтобы расширение соблюдало законы и правила конфиденциальности.  Вы также несете ответственность за предоставление URL-адреса политики конфиденциальности, если какие-либо персональные данные передаются или собираются расширением.  Чтобы определить, требуется ли вашему расширению политика конфиденциальности, перейдите к [соглашению разработчиков Microsoft Edge][MicrosoftAppDeveloperAgreement] и [microsoft Edge Add-ons store developer policies.][MicrosoftEdgeAddonsCatalogDeveloperPolicies]  |  
| URL-адрес веб-сайта | Веб-страница, которая предоставляет дополнительные сведения о расширении.  Необходимо указать на страницу на собственном сайте, а не на веб-лист для расширения в `Website URL` магазине надстройок Microsoft Edge.  Это `Website URL` помогает пользователям узнать больше о расширении, его особенностях и любой другой соответствующей информации.  |  
| Поддержка контактных данных | URL-адрес веб-страницы поддержки или адрес электронной почты, чтобы связаться с вашей командой поддержки.  |  
| Зрелый контент | Контрольный ящик, чтобы указать, включает ли ваше расширение зрелое содержимое.  Оценка расширения помогает определить соответствующую возрастную группу целевой аудитории вашего расширения.  Чтобы определить, имеет ли ваше расширение зрелое содержимое, перейдите в [microsoft Edge Add-ons store developer policies.][MicrosoftEdgeAddonsCatalogDeveloperPolicies]  |  

Выберите **Сохранить,** чтобы продолжить работу в **разделе Списки магазина.**  

> [!Important]
> Имя разработчика или организации, URL-адрес веб-сайта и контактные данные поддержки, которые были представлены во время регистрации, отображаются пользователям в магазине надстройок Microsoft Edge.  

#### <a name="step-5-add-store-listing-details-for-your-extension"></a>Шаг 5. Добавление сведений о листинге магазина для расширения  

Сведения, предоставленные в следующем разделе, отображаются пользователям, которые просматривают ваше объявление в магазине надстройок Microsoft Edge.  Несмотря на то, что некоторые поля необязательны, необходимо предоставить как можно больше информации.  Чтобы перечислить расширение в магазине, необходимы следующие сведения.  

*   **Описание** для каждого языка в пакете расширения.  
*   **Логотип Магазина расширения** для каждого языка в пакете расширения.  
    
> [!NOTE]
> Минимальные необходимые сведения о листинге магазина должны быть заполнены по крайней мере для одного из языков, указанных в пакете почтовых индексов расширения.  Чтобы добавить или удалить языки в списке магазина в магазине **** надстройок Microsoft Edge, используйте страницу Добавить язык на странице **Списки** магазинов.  Кроме того, вы можете дублировать свои активы с одного языка на других с помощью кнопки дубликата функций на странице языковые сведения.  

| Имя свойства языковых сведений | Описание |  
|:--- |:--- |  
| Отображение имени \(required\) | Расширение, `name` указанное в файле манифеста вашего расширения.  Чтобы изменить имя отображения магазина после отправки, можно обновить имя в файле манифеста, создать новый пакет расширений и затем перезагрузить его.  |  
| Описание \(обязательно\) | В поле основное внимание уделяется объяснению того, что делает ваше расширение, почему пользователи должны установить его или другие релевантные сведения, которые `description` необходимо знать пользователям.  Она должна быть менее 10 000 символов.  |  
| Логотип магазина расширения \(required\) | Изображение, представляюное вашу компанию или с соотношением аспектов 1 и рекомендуемый размер `extension logo` 300 x 300 пикселей.  Кроме того, вы можете скопировать актив с одного языка на все другие языки с помощью кнопки дубликата.  Кнопка находится в поле после отправки логотипа для языка.  |  
| Небольшая рекламная плитка \(необязательный\) | Изображение используется для отображения расширения наряду с `Small promotional tile` другими расширениями в магазине.  Размер изображения должен быть 440 x 280 пикселей.  Кроме того, вы можете скопировать актив с одного языка на все другие языки с помощью кнопки дубликата.  Кнопка находится в поле после отправки рекламной плитки для языка.  |  
| Скриншоты \(необязательный\) | Вы можете представить не более 10 подробных описаний функциональных `screenshots` возможностей расширения.  Размер скриншотов должен быть 640 x 480 пикселей или 1280 x 800 пикселей.  Кроме того, вы можете скопировать актив с одного языка на все другие языки с помощью кнопки дубликата.  Кнопка будет найдена после отправки по крайней мере одной кнопки для языка.|  
| Большая рекламная плитка \(необязательный\) | `Large promotion tiles` используются в магазине для более заметного использования расширений на веб-сайте надстройок Microsoft Edge.  Изображения, если они будут представлены, будут видны пользователям.  Размер PNG-файлов должен быть 1400 x 560 пикселей.  Кроме того, вы можете скопировать актив с одного языка на все другие языки с помощью кнопки дубликата.  Кнопка находится в поле после отправки рекламной плитки для языка.  |  
| URL-адрес видео YouTube \(необязательный\) | Вы можете включить рекламный видео YouTube вашего расширения.  Видео `YouTube video URL` отображается на странице списка магазина вашего расширения.  |  
| Краткое описание \(required\) | Чтобы изменить поле описания, необходимо обновить поле описания в файле манифеста пакета расширения и перезагрузить `short description` его.  |  
| Условия поиска \(необязательный\) | `Search terms` это отдельные слова или фразы, которые помогают пользователям обнаружить расширение при поиске в магазине надстройок Microsoft Edge.  Термины поиска не отображаются пользователям.  |  

##### <a name="youtube-video-url-requirements"></a>Требования к URL-адресу видео youTube  

Убедитесь, что видео соответствует следующим требованиям.  

*   Убедитесь, что содержимое видео На YouTube следует [политике разработчика Магазина надстройок Microsoft Edge.][MicrosoftEdgeAddonsCatalogDeveloperPolicies]  
*   Отключите рекламу на видео.  Дополнительные сведения можно найти в настройках [форматов][GoogleYoutubeAnswer2531367Topic7072227] объявлений по умолчанию и [объявлений на встроенных видео.][GoogleYoutubeAnswer132596]  
*   Включите встраивку для видео.  Дополнительные сведения см. в [встраив видео & плейлисты][GoogleYoutubeAnswer171780].  
    
Чтобы отправить URL-адрес видео на YouTube, выполните следующие действия.  

1.  На YouTube найдите видео, которое необходимо добавить на страницу списка магазинов.  
1.  В видео выберите **Share**  >  **Embed**.  
1.  Скопируйте отображаемый HTML-код.  
1.  На странице сведения о листинге магазина вклейте HTML-код в `YouTube video URL` поле.  
    
##### <a name="search-terms-requirements"></a>Требования к условиям поиска  

Условия поиска должны соответствовать следующим требованиям.  

*   Вы можете ввести термины поиска, чтобы использовать не более 21 слова.  Используется ли в качестве одного слова, фраз или комбинации обоих слов, разрешено использовать не более 21 слова.  
*   Не более семи терминов поиска: одно слово или фразы.  Каждый термин поиска имеет ограничение символов в 30 символов.  
    
#### <a name="step-6-complete-your-submission"></a>Шаг 6. Завершение отправки  

На странице **Отправка расширения** добавьте заметки для сертификации, чтобы проверить расширение.  

##### <a name="notes-for-certification-optional"></a>Заметки для сертификации (необязательный)  

При отправке расширения используйте страницу **Заметки для** сертификации для предоставления дополнительных сведений для тестеров сертификации.  Дополнительные сведения помогают убедиться в правильности проверки расширения.  Если расширение не полностью протестировано, сертификация может привести к сбойу.  

Убедитесь, что при необходимости включаем следующие сведения.  

*   Имена пользователей и пароли для тестовых учетных записей.  
*   Действия для доступа к скрытым или заблокированным функциям.  
*   Ожидаемые различия в функциональных возможностях в зависимости от региона или других параметров пользователя.  
*   Если отправка является обновлением существующего расширения, включите сведения об изменениях, внесенных в расширение.  
*   Любые дополнительные сведения, которые тестеры должны понимать о вашей отправке.  

После предоставления сведений выберите **Опубликовать,** чтобы отправить расширение в магазин надстройок Microsoft Edge.  Отправка переходит к этапу сертификации.  Процесс сертификации может занять до семи бизнес-дней после отправки.  

После сертификации отправки расширение публикуется в магазине надстройок Microsoft Edge.  Состояние расширения в панели мониторинга Центра партнеров изменяется на `In the Store` .  

> [!NOTE]
> Если в процессе отправки или регистрации возникнут какие-либо проблемы, отправьте заявку на поддержку в [Новом][ExtensionsSupportForm] запросе на поддержку расширения или отправьте электронное сообщение в [ext_dev_support@microsoft.com.][MailtoExtDevSupportMicrosoftCom]
