# Публикация расширения  

После разработки и тестирования расширения вы можете распространять расширение. Используйте магазин надстройки Microsoft Edge для распространения расширения.  Чтобы освободить существующее расширение Chromium для пользователей Microsoft Edge, перейдите к переносу существующего расширения [Chromium.][PortChromiumExtension]  

Опубликуйте расширение в магазине надстройки Microsoft Edge, чтобы увеличить его охват и сделать его доступным для пользователей Microsoft Edge.  В этой статье приводится процесс отправки расширения в магазин надстройки Microsoft Edge.  

## Перед началом работы  

У вас должен быть готовый рабочий прототип расширения.  Сведения о создании расширения можно найти в учебнике по [началу работы.][ExtensionsGettingStarted]  

Чтобы опубликовать расширение в магазине надстройки Microsoft Edge, используйте свою активную учетную запись разработчика в [Центре партнеров.][MicrosoftPartnerCenter]  Если у вас нет учетной записи разработчика, создайте новую учетную запись разработчика.  Чтобы открыть новую учетную запись разработчика и зарегистрироваться в программе надстройки Microsoft Edge, перейдите к регистрации [разработчика.][DeveloperRegistration]  

Создайте ZIP-файл, который представляет ваш пакет расширения.  Пакет расширения должен включать следующие файлы.  

*   Манифест расширения, определяющий такие сведения, как имя расширения, краткое описание, разрешения и язык по умолчанию.  
*   Изображения и другие файлы, необходимые для расширения.  

Следующие поля манифеста автоматически включаются в описание в Магазине.  Поля на странице "Описание в **Магазине" будут только для** чтения.  Страница описаний в Магазине описана далее в этой статье.  Перед отправкой пакета в Центр партнеров убедитесь, что значения полей соответствуют предпочитаемой странице сведений о магазине.  В качестве примера кода, необходимого для файла манифеста, просмотрите основные принципы файла манифеста.  

*   `Name` поле в файле манифеста, которое является **отображаемой** именем на странице сведений о хранилище.  
*   `Description` поле в файле манифеста, который является **кратким** описанием на странице сведений о хранилище.  Укажийте краткое описание, отображаемое в верхней части описания расширения.  Если этот файл включен, краткое описание, указанное в файле манифеста расширения, отображается в описании в Магазине.  Если краткое описание не включено в файл манифеста, отображаются первые несколько строк описания.  Укажийте краткое описание, чтобы избежать повторения содержимого на странице описания в Магазине.  

## Отправка расширения в магазин надстройки Microsoft Edge  

Чтобы отправить расширение в [Центр партнеров,][MicrosoftPartnerCenter]с помощью следующих действий.  

#### Шаг 1. Запуск новой отправки  

Перейдите на [информационную панель разработчика][MicrosoftPartnerCenter] и выберите **"Создать новое расширение"** на странице **"Обзор".**  

#### Шаг 2. Отправка пакета расширения  

Используйте **страницу "Пакеты"** для отправки ZIP-файла пакета расширения.  Вы можете отправить только один пакет за раз.  Вы не сможете продолжить отправку, если отправка пакета не будет успешной на странице **"Пакеты".**  

Чтобы отправить пакет, выберите и перетащите пакет в поле отправки. Вы также можете выбрать **"Обзор файлов".**  После отправки пакета пакет проверяется.  После успешной проверки просмотрите сведения о **** расширении и выберите "Далее", чтобы продолжить.  Если имеются ошибки проверки, улажийте проблемы и попробуйте отправить их еще раз.  

#### Шаг 3. Предоставление сведений о доступности  

На странице **"Доступность"** введите следующие сведения о доступности расширения.  

##### Видимость  

Выберите один из следующих вариантов видимости, чтобы определить, можно ли обнаружить расширение в магазине надстройки Microsoft Edge.  

*   `Public` \(default\)  
    Общедоступный позволяет всем обнаруживать ваше расширение с помощью поиска, просмотра в магазине надстройки Microsoft Edge или использования URL-адреса с описанием расширения в магазине надстройки Microsoft Edge.  URL-адрес списка доступен на информационной панели Центра партнеров на странице **"Обзор расширения".**  
    
*   `Hidden`  
    Скрытые расширения удаляются из результатов поиска или просмотра в магазине надстройки Microsoft Edge.  Для распространения скрытых расширений в магазине надстройки Microsoft Edge необходимо поделиться URL-адресом с описанием расширения с вашими клиентами.  
    
> [!NOTE]
> Вы можете изменить видимость расширения с **"Общедоступный"** на **"Скрытый".**  Пользователи, которые установили расширение, когда для видимости было установлено открытое сохранение доступа к вашему расширению и получение всех обновлений, доступных на веб-сайте надстройки Microsoft Edge.  

##### Страны  

Определите конкретные рынки, на которых планируется предлагать расширение.  Параметром по умолчанию для рынков являются все рынки, в том числе любые будущие рынки, которые будут добавлены позже.  Чтобы выбрать определенные рынки, выберите **"Изменить рынки".**  Чтобы исключить отдельные рынки, **** выберите "Отбрать все" и добавить отдельные рынки по вашему выбору.  

> [!NOTE]
> Вы можете изменить рынки, на которых предлагается расширение.  Пользователь, который устанавливает расширение, пока оно доступно на рынке пользователя, сохраняет доступ к вашему расширению.  Однако у пользователя нет доступа к будущим обновлениям, отправленным в магазин надстройки Microsoft Edge.  

Выберите **"Сохранить",** чтобы продолжить работу с **разделом "Свойства".**  

#### Шаг 4. Выбор свойств для расширения  

На странице **"Свойства"** введите следующие сведения, чтобы указать свойства расширения.  Свойства отображаются для пользователей в магазине надстройки Microsoft Edge.  

| Имя свойства расширения | Описание |  
|:--- |:--- |  
| Category \(required\) | Категория, которая лучше всего описывает расширение.  Перечисление расширения в нужной категории поможет пользователям легко найти расширение и понять его подробнее.  |  
| Требования политики конфиденциальности \(обязательно\) | Указать, имеет ли расширение доступ, собирает или передает какие-либо персональные данные.  Если вы выберете **"Да"** и не предоставите сертификат, расширение может не быть `Privacy policy URL` сертифицировано.  |  
| URL-адрес с политикой конфиденциальности | Допустимый URL-адрес политики конфиденциальности, чтобы сообщить, как расширение соблюдает законы и нормы конфиденциальности.  Вы несете ответственность за то, чтобы расширение соблюдало законы и нормы конфиденциальности.  Вы также несете ответственность за предоставление URL-адреса политики конфиденциальности, если ваши расширения предоставляют доступ к какой-либо личной информации, передаются или собираются.  Чтобы определить, требуется ли вашему расширению политика конфиденциальности, перейдите к [соглашению][MicrosoftAppDeveloperAgreement] с разработчиком Microsoft Edge и политикам разработчика надстройки [Microsoft Edge.][MicrosoftEdgeAddonsCatalogDeveloperPolicies]  |  
| URL-адрес веб-сайта | Веб-страница, которая предоставляет дополнительные сведения о расширении.  Необходимо указать страницу на вашем веб-сайте, а не на веб-страницу с описанием вашего расширения в магазине `Website URL` надстройки Microsoft Edge.  Это `Website URL` помогает пользователям узнать больше о расширении, его функции и любые другие важные сведения.  |  
| Контактные данные службы поддержки | URL-адрес веб-страницы поддержки или адрес электронной почты для обращения в службу поддержки.  |  
| Сконтентное содержимое | Checkbox to specify if your extension includes mature content.  Оценка расширения помогает определить соответствующую возрастную группу целевой аудитории вашего расширения.  Чтобы определить, имеется ли у расширения содержимое, перейдите к политикам разработчика магазина надстройки [Microsoft Edge.][MicrosoftEdgeAddonsCatalogDeveloperPolicies]  |  

Выберите **"Сохранить",** чтобы продолжить работу с **разделом "Описание в** Магазине".  

> [!Important]
> Имя разработчика или организации, URL-адрес веб-сайта и контактные данные службы поддержки, которые вы отправили во время регистрации, отображаются пользователям в магазине надстройки Microsoft Edge.  

#### Шаг 5. Добавление сведений о списке в Магазине для расширения  

Сведения, предоставленные в следующем разделе, отображаются для пользователей, просматривавших ваше описание в магазине надстройки Microsoft Edge.  Хотя некоторые поля являются необязательными, необходимо предоставить как можно больше информации.  Чтобы получить список расширений в Магазине, необходимы следующие сведения.  

*   **Описание** каждого языка в пакете расширения.  
*   **Логотип Магазина расширений** для каждого языка в пакете расширения.  
    
> [!NOTE]
> Минимальные необходимые сведения о списке в магазине должны быть заполнены по крайней мере для одного из языков, упомянутых в ZIP-пакете расширения.  Чтобы добавить или удалить языки в списке магазина в магазине **** надстройки Microsoft Edge, используйте в этом выпадаемом списке на странице "Описание **в Магазине".**  Кроме того, вы можете дублировать ресурсы с одного языка на другой с помощью кнопки дублирования функций на странице сведений о языке.  

| Имя свойства сведений о языке | Описание |  
|:--- |:--- |  
| Отображаемая имя \(обязательно\) | Расширение, `name` указанное в файле манифеста расширения.  Чтобы изменить отображаемую папку магазина после отправки, можно обновить имя в файле манифеста, создать новый пакет расширения, а затем повторно его перезагрузить.  |  
| Description \(required\) | В этом поле основное внимание уделяется объяснению того, что делает расширение, почему пользователи должны установить его, или другой важной информации, `description` необходимой пользователям.  Оно должно быть меньше 10 000 символов.  |  
| Логотип магазина расширений \(обязательно\) | Изображение, которое представляет вашу компанию или с пропорциями 1 и рекомендуемый размер `extension logo` 300 x 300 пикселей.  Кроме того, вы можете скопировать актив с одного языка на все другие языки с помощью кнопки дубликата.  Кнопка находится после поля после отправки логотипа для языка.  |  
| Небольшая рекламная плитка \(необязательно\) | Изображение используется для отображения расширения вместе с `Small promotional tile` другими расширениями в магазине.  Размер изображения должен быть 440 x 280 пикселей.  Кроме того, вы можете скопировать актив с одного языка на все другие языки с помощью кнопки дубликата.  Кнопка находится после поля после отправки рекламной плитки для языка.  |  
| Screenshots \(optional\) | Вы можете отправить не более 10 с подробным описанием функциональности `screenshots` расширения.  Размер снимков экрана должен быть 640 x 480 пикселей или 1280 x 800 пикселей.  Кроме того, вы можете скопировать актив с одного языка на все другие языки с помощью кнопки дубликата.  Кнопка будет найдена после отправки по крайней мере одной кнопки для языка.|  
| Большая рекламная плитка \(необязательно\) | `Large promotion tiles` используются в Магазине для расширения функций на веб-сайте надстройки Microsoft Edge.  При отправке изображения видны пользователям.  Размер PNG-файлов должен быть 1400 x 560 пикселей.  Кроме того, вы можете скопировать актив с одного языка на все другие языки с помощью кнопки дубликата.  Кнопка находится после поля после отправки рекламной плитки для языка.  |  
| URL-адрес видео YouTube \(необязательно\) | You may include a promotional YouTube video of your extension.  Видео отображается на странице с описанием вашего расширения `YouTube video URL` в Магазине.  |  
| Краткое описание \(обязательно\) | Чтобы изменить поле описания, необходимо обновить поле описания в файле манифеста пакета расширения и `short description` перезагрузить его.  |  
| Условия поиска \(необязательно\) | `Search terms` это отдельные слова или фразы, которые помогают пользователям обнаружить расширение при поиске в магазине надстройки Microsoft Edge.  Условия поиска не отображаются для пользователей.  |  

##### Требования к URL-адресам видео youTube  

Убедитесь, что видео соответствует следующим требованиям.  

*   Убедитесь, что содержимое видеоролика на YouTube следует политикам разработчика для надстройки [Microsoft Edge.][MicrosoftEdgeAddonsCatalogDeveloperPolicies]  
*   Отключите рекламу в видео.  Для получения дополнительных сведений перейдите к [настройкам форматов][GoogleYoutubeAnswer2531367Topic7072227] объявлений по умолчанию и [рекламы для внедренных видео.][GoogleYoutubeAnswer132596]  
*   Включите встраивку для своих видео.  Для получения дополнительных сведений перейдите к [встраив видео & списки воспроизведения][GoogleYoutubeAnswer171780].  
    
Чтобы отправить URL-адрес видеоролика youTube, выполните следующие действия.  

1.  На YouTube найдите видео, которое вы хотите добавить на страницу с описанием в Магазине.  
1.  В видео выберите **"Поделиться**  >  **встраить".**  
1.  Скопируйте отображаемый HTML-код.  
1.  На странице сведений о листинге в магазине в поле в поле в paste `YouTube video URL` htmL-код.  
    
##### Требования к условиям поиска  

Условия поиска должны соответствовать следующим требованиям.  

*   Вы можете ввести условия поиска, чтобы использовать не более 21 слова.  Независимо от того, используются ли отдельные слова, фразы или их сочетание, допускается не более 21 слова.  
*   Не более семи поисковых терминов: одно слово или фразы.  Каждый термин поиска имеет ограничение в 30 символов.  
    
#### Шаг 6. Завершение отправки  

На странице **"Отправка расширения"** добавьте заметки для сертификации, чтобы проверить расширение.  

##### Примечания по сертификации (необязательно)  

При отправке расширения используйте страницу **"Заметки** для сертификации", чтобы предоставить дополнительные сведения тест-испытателям сертификации.  Дополнительные сведения помогают проверить правильность проверки расширения.  Если расширение не полностью протестировано, оно может не проходить сертификацию.  

При необходимости включаем следующие сведения.  

*   Имена пользователей и пароли тестовых учетных записей.  
*   Действия для доступа к скрытым или заблокированным функциям.  
*   Ожидаемые различия в функциональных возможностях в зависимости от региона или других пользовательских параметров.  
*   Если отправка является обновлением существующего расширения, включите сведения об изменениях, внесенных в расширение.  
*   Любая дополнительная информация, которую должны знать тест-тестеры о вашей отправке.  

После предоставления сведений выберите **"Опубликовать",** чтобы отправить расширение в магазин надстройки Microsoft Edge.  Отправка переходит к этапу сертификации.  Процесс сертификации может занять до семи бизнес-дней после отправки.  

Когда отправка пройдет сертификацию, расширение будет опубликовано в магазине надстройки Microsoft Edge.  Состояние расширения на информационной панели Центра партнеров изменится на `In the Store` .

> Если в процессе отправки или регистрации возникнут проблемы, отправьте запрос в службу поддержки по запросу на поддержку [extensions New Support Request][ExtensionsSupportForm] или отправьте сообщение электронной почты [ext_dev_support@microsoft.com.][MailtoExtDevSupportMicrosoftCom]
