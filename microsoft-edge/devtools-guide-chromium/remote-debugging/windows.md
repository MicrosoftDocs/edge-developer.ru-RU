# Начало работы с удаленной отладкой устройств с Windows 10  

Удаленная отладка динамического содержимого на устройстве с Windows 10 с компьютера с Windows или macOS.  В этом учебнике рассказывается о том, как выполнять следующие действия:  

*   Настройте устройство с Windows 10 для удаленной отладки и подключите его с компьютера разработчика.  
*   Изучите и отлаживать динамический контент на устройстве с Windows 10 с компьютера для разработки.  
*   Содержимое ролика с устройства с Windows 10 на экземпляр DevTools на компьютере разработчика.  

## Шаг 1: Настройка узла (для отлаживаемого компьютера)  

Хостом или отлаживаемым компьютером является устройство Windows 10, которое вы хотите отладить.  Возможно, это удаленное устройство, к которому вы можете получить доступ, или у него нет клавиатуры и мыши, что затрудняет взаимодействие с Microsoft Edge DevTools на этом устройстве.  Для настройки машины Host (отлаживаемая программа) необходимо:  

*   Установка и настройка [Microsoft EDGE (Chromium)](https://www.microsoft.com/edge)  
*   Установка [средств удаленного доступа для Microsoft EDGE (бета-версия)](https://www.microsoft.com/store/apps/9P6CMFV44ZLT) из [Microsoft Store](https://www.microsoft.com/store/apps/windows)  
*   Активация [режима разработчика](https://docs.microsoft.com/windows/uwp/get-started/enable-your-device-for-development) и включение [портала устройств](https://docs.microsoft.com/windows/uwp/debug-test-perf/device-portal)  

### Установка и настройка Microsoft EDGE (Chromium)  

Если вы еще не сделали этого, установите Microsoft EDGE (Chromium) на [этой странице](https://www.microsoft.com/edge).  Если вы используете предварительно установленную версию Microsoft EDGE на компьютере Host (отлаживаемой стороне), убедитесь в том, что у вас есть Microsoft EDGE (Chromium), а не Microsoft EDGE (EdgeHTML).  Чтобы быстро проверить, нужно ли загрузить `edge://settings/help` браузер и убедиться, что номер версии — 75 или выше.  

Теперь перейдите `edge://flags` в Microsoft EDGE (Chromium).  В списке **Флаги поиска**введите **включить удаленную отладку через портал устройств Windows**.  Установите этот флажок для **включения**.  Затем нажмите кнопку **перезапустить** , чтобы перезапустить Microsoft EDGE (Chromium).

Обновите компьютер узла (отлаживаемого кода), чтобы получить [удаленные инструменты для Microsoft EDGE (Beta)](https://www.microsoft.com/store/apps/9P6CMFV44ZLT).

### Активация режима разработчика и включение портала устройств  

Если вы находитесь в сети Wi-Fi, убедитесь, что сеть помечена как **domain** или **Private**.  Вы можете проверить это, открыв приложение **безопасности Windows** , щелкнув **брандмауэр & защиту сети** и проверяя, отображается ли сеть в виде **доменной** сети или **частной** сети.  

Если он указан как **общедоступный**, перейдите в раздел **Параметры**  >  **& сети**  >  **Wi-Fi**, щелкните свою сеть и переключитесь на кнопку **Network Profile** ( **Частная**).  

Теперь откройте приложение **Параметры** .  В окне **найти параметр**введите **Параметры разработчика** и выберите его.  Включить или выключить **режим разработчика**.  Теперь можно включить **портал устройств** , включив параметр **включить удаленную диагностику через локальную сеть** . **On**  Вы можете дополнительно включить **проверку подлинности** , чтобы клиент (отладчик) предоставил вам правильные учетные данные для подключения к этому устройству.  

> [!NOTE]
> Если **включена удаленная диагностика для подключений по локальной сети.** была включена, вы должны отключить и снова включить ее на **портале устройств** для работы с [удаленными инструментами для Microsoft EDGE (Beta)](https://www.microsoft.com/store/apps/9P6CMFV44ZLT). Если вы не видите раздел " **для разработчиков** " в разделе " **Параметры**", возможно, **портал устройств** уже включен, поэтому попробуйте перезагрузить устройство с Windows 10.

## Шаг 2: Настройка клиента (компьютера отладчика)  

Клиент или компьютер отладчика — это устройство, с которого вы хотите выполнить отладку.  Это может быть ваш ежедневный компьютер для разработки, или это может быть только ваш компьютер или MacBook, если вы работаете дома.  

Чтобы настроить клиентский компьютер (отладчик), установите Microsoft EDGE (Chromium) с [этой страницы](https://www.microsoft.com/edge) , если вы еще этого не сделали.  Если вы используете предварительно установленную версию Microsoft EDGE на компьютере Host (отлаживаемой стороне), убедитесь в том, что у вас есть Microsoft EDGE (Chromium), а не Microsoft EDGE (EdgeHTML).  Чтобы быстро проверить, нужно ли загрузить `edge://settings/help` браузер и убедиться, что номер версии — 75 или выше.  

Теперь перейдите `edge://flags` в Microsoft EDGE (Chromium).  В области **Флаги поиска**введите в **Edge://inspect включить удаленную отладку устройств Windows**.  Установите этот флажок для **включения**.  Затем нажмите кнопку **перезапустить** , чтобы перезапустить Microsoft EDGE (Chromium).  

> ##### Рисунок 5  
> Настройка функции " **включить отладку удаленного устройства Windows с помощью** флага Edge://inspect" для **включения**  
> ![Настройка функции "включить отладку удаленного устройства Windows с помощью флага edge://inspect" для включения](./windows-media/edge-flags-on-client.png)  

Теперь перейдите на `edge://inspect` страницу в Microsoft EDGE (Chromium).  По умолчанию вы должны находиться в разделе " **устройства** ".  В разделе **Подключение к удаленному устройству Windows**введите IP-адрес и порт (отлаживаемый объект) Host (отлаживаемая) компьютера в текстовом поле, следующем шаблоне: http:// `IP address` : `connection port` .  Теперь нажмите кнопку **подключиться к устройству**.  

> ##### Рисунок6  
> `edge://inspect`Страница на стороне клиента

Если вы настроили проверку подлинности для компьютера узла (отлаживаемого объекта), вам будет предложено ввести **имя пользователя** и **пароль** для клиентского компьютера (отладчика) для успешной подсоединения.  

### Использование HTTPS вместо HTTP  

Если вы хотите подключиться к компьютеру с отлаживаемым приложением `https` , используя вместо этого `http` , вы должны Navgiate в `http://IP address:50080/config/rootcertificate` Microsoft EDGE на клиентском компьютере (отладчике). Будет автоматически загружен сертификат безопасности с именем `rootcertificate.cer` .

Нажмите кнопку вкл `rootcertificate.cer` . Откроется [средство управления сертификатами Windows](https://docs.microsoft.com/dotnet/framework/wcf/feature-details/how-to-view-certificates-with-the-mmc-snap-in#view-certificates-with-the-certificate-manager-tool).

Нажмите кнопку **установить сертификат..**., убедитесь, что выбран **текущий пользователь** , и нажмите кнопку **Далее**. Затем выберите **поместить все сертификаты в следующее хранилище** и нажмите кнопку **Обзор..**.. Выберите хранилище **Доверенные корневые центры сертификации** и нажмите кнопку **ОК**. Нажмите кнопку **Далее**, а затем — кнопку **Готово**. При появлении запроса подтвердите, что вы хотите установить этот сертификат в хранилище **доверенных корневых центров сертификации** .

Теперь, когда вы подключаетесь к компьютеру с отлаживаемым приложением с клиентского компьютера (отладчика) с помощью `edge://inspect` страницы, необходимо использовать другое `connection port` значение.  По умолчанию для настольных окон портал устройств будет использоваться `50080` в качестве " `connection port` for" `http` .  Для этого на `https` портале устройств используются `50043` следующие шаблоны: https:// `IP address` : `50043` на `edge://inspect` странице.  [Узнайте больше о портах по умолчанию, используемых порталом устройств](https://docs.microsoft.com/windows/uwp/debug-test-perf/device-portal#setup).  

> [!NOTE]
> По умолчанию для порта используется порт по умолчанию, `http` `50080` `https` `50043` но это не всегда так же, как портал устройств на рабочем столе (>50 000), чтобы предотвратить конфликты с существующими утверждениями порта на устройстве.  Дополнительные сведения можно найти в разделе [Параметры порта](https://docs.microsoft.com/windows/uwp/debug-test-perf/device-portal-desktop#registry-based-configuration-for-device-portal) на портале устройств на рабочем столе Windows.  

## Шаг 3: Отладка содержимого на узле из клиента  

Если клиент (отладчик) успешно подключается к компьютеру Host (отлаживаемая программа), на `edge://inspect` странице клиента будет отображен список вкладок в Microsoft EDGE и все открытые PWAs на узле.

Определите содержимое, которое вы хотите отладить, и нажмите кнопку **проверить**.  Microsoft Edge DevTools откроется на новой вкладке и выведет содержимое с компьютера хоста (отлаживаемой программы) на клиентский компьютер (отладчик).  Теперь вы можете использовать полную мощь Microsoft Edge DevTools на клиенте для содержимого, запущенного на узле.  Узнайте больше о том, как использовать Microsoft [Edge DevTools.](../../devtools-guide-chromium.md)

### Проверка элементов  

Например, попробуйте проверить элемент.  Перейдите на панель **элементы** своего экземпляра DevTools на клиентском компьютере и наведите указатель мыши на элемент, чтобы выделиь его в окне просмотра основного устройства.  

Вы также можете коснуться элемента на экране хоста устройства, чтобы выбрать его на панели " **элементы** ".  Нажмите кнопку **выбрать элемент** в экземпляре DevTools на клиентском компьютере, а затем коснитесь элемента на экране хоста устройства.  Обратите внимание, что **элемент SELECT** отключается после первого касания, поэтому при каждой попытке использования этой функции необходимо повторно включить его.  

> [!IMPORTANT]
> Область " **прослушиватели событий** " на панели " **элементы** " пуста в Windows 10 версии 1903.  Это известная проблема, и мы исправим область " **прослушиватели событий** " в обновлении обслуживания до Windows 10 версии 1903.  

## Шаг 4: демонстрация экрана размещения на клиентском устройстве  

По умолчанию для экземпляра DevTools на клиентском компьютере будет включена возможность показа презентации, что позволяет просматривать содержимое на хостном устройстве в экземпляре DevTools на клиентском устройстве.  Нажмите кнопку **Переключить** на экранную презентацию, чтобы отключить или включить эту функцию.  

> ##### Рисунок9  
> Кнопка " **Переключить презентацию** " в Microsoft Edge DevTools на стороне клиента  
> ![Кнопка "переключить презентацию" в Microsoft Edge DevTools на стороне клиента](./windows-media/toggle-screencast.png)  

Вы можете работать с презентацией несколькими способами.  
*   Нажатия клавиш переводятся в касания, и на устройстве обрабатывались соответствующие события сенсорного ввода.  
*   Нажатия клавиш на компьютере отправляются на устройство.  
*   Чтобы смоделировать жест сжатия, удерживайте `Shift` при перетаскивании.  
*   Для прокрутки используйте сенсорной панели, колесико мыши или смахивание с помощью указателя мыши.  

Некоторые заметки для презентаций:  
*   В ролики отображается только содержимое страницы.  Прозрачные части презентации представляют интерфейсы устройства, такие как строка адреса Microsoft EDGE, панель задач Windows 10 или клавиатура Windows 10.  
*   Ролики негативно влияют на частоту кадров.  Отключите экранную анимацию, изменяя прокрутки или анимации, чтобы получить более точную картину производительности страницы.  
*   Если экран вашего хоста блокируется, содержимое презентации исчезнет.  Разблокируйте экран устройства хоста для автоматического возобновления ролика.  

## Известные проблемы  

Область " **прослушиватели событий** " на панели " **элементы** " пуста в Windows 10 версии 1903.  Мы исправим область " **прослушиватели событий** " в обновлении обслуживания до Windows 10 версии 1903.  

Область **"cookies** " на панели **приложения** пуста в Windows 10 версии 1903.  Область **cookies** будет исправлена в обновлении обслуживания до Windows 10 версии 1903.  

Панель " **Аудит** ", панель **трехмерного представления** , **эмулированные устройства** в разделе " **Параметры**" и область **дерева специальных возможностей** на панели " **элементы** " в настоящее время не работают должным образом.  Эти средства будут исправлены в следующем обновлении Microsoft Edge.  

При удаленной отладке проводник не запускается из DevTools на панели " **источники** " или на панели " **Безопасность** ".  Эти средства будут исправлены в следующем обновлении Microsoft Edge.
