# Трехмерное представление  

С помощью **трехмерного представления** вы можете выполнить отладку вашего веб-приложения, перейдя по [объектной модели документов (DOM)][MDNDocumentObjectModel] или в контекст стека [z-индексов][MDNZIndex] .  С его помощью вы можете выполнить указанные ниже действия.  

*   [Знакомство с веб-страницей, преобразованной в трехмерную перспективу](#3d-dom)  
*   [Отладка с учетом контекста z-индексов](#z-index)  
*   [Доступ к функциям слоев из трехмерного представления с составными слоями](#composited-layers)  
*   [Очистка области "несрочные" на панели DOM](#changing-your-view) или [области "z-указатель](#change-the-scope-of-your-exploration) "  
*   [Выбор цветовой схемы для лучшей отладки проблем с DOM](#dom-color-type) и [проблем с z-индексом](#z-index-color-type)  

Если вы хотите ознакомиться с ранним прототипом проекта 3D View и выполнить код самостоятельно, перейдите к [примеру 3D View][GithubMicrosoftedgeDevtoolssamples3dview].  

В левой части экрана есть три области, которые можно использовать для работы с отладчиками.  

*   Область " [Z-индекс](#z-index) ".  Просмотрите различные элементы веб-приложения с помощью контекста z-индексов.  Область " **Z-индекс** " — область "по умолчанию".  
*   Область [3D DOM](#3d-dom) .  Изучите модель DOM целиком, с помощью которой можно легко получить доступ к элементам.  Для доступа к области выберите область **DOM** рядом с областью " **Z-индекс** ".  
*   Область « [составной слой](#composited-layers) ».  Добавьте еще один трехмерный элемент, чтобы создать более полный интерфейс с точки зрения слоев.  Чтобы открыть область, выберите область **Составные слои** рядом с областью **DOM** .  
    
В правой части холста отображаются выбранные значения из [Z-Index](#z-index), [3D DOM](#3d-dom)или [составных слоев](#composited-layers).

### Сочетания клавиш  

*   Поворот модели DOM: чтобы повернуть ее по горизонтали, выберите `left-arrow` `right-arrow` клавиши и.  Чтобы повернуть по вертикали, `up-arrow` выберите `down-arrow` клавиши и.  
*   Навигация по модели DOM: для перемещения по смежным элементам выберите элемент и выберите нужные `up-arrow` `down-arrow` Параметры.  

### Элементы управления мышью  

*   Поворот DOM: выбор и перетаскивание вокруг области холста.  
*   Сдвиг на модель DOM: Откройте контекстное меню, щелкните правой кнопкой мыши и перетащите указатель в том направлении, в котором вы хотите переместить модель DOM.  
*   Zoom (масштаб): перетащите два пальца на сенсорной панели или используйте колесо прокрутки на мыши.

## Z-индекс  

:::image type="complex" source="../media/3d-view-z-index-view-box.msft.png" alt-text="Представление "Z-индекс"" lightbox="../media/3d-view-z-index-view-box.msft.png":::
   Представление "Z-индекс"  
:::image-end:::  

Несмотря на то, что в области **Z-индекс** есть общие функции с областью **трехмерной модели DOM** , в областях остались элементы, которые являются уникальными для области.  

### Выделение элементов с помощью контекста размещения  

**Элементы "выделение" с контекстным** параметром позволяют включить \ (и отключить) Теги z-index для элементов на холсте.  По умолчанию выбран флажок.  

### Изменение области исследования  

Кнопка " **Показать все элементы** " — самый быстрый способ отобразить все элементы DOM после изменения параметров, расположенных ниже.  

Кнопка **Показать только элементы с контекстным** окном удаляет элементы без контекста размещения и ОБРАВНИВАЮТ модель DOM для упрощения навигации.  

Кнопка « **изолировать выделенный элемент** » — по сути три кнопки в одной.  Рядом с кнопкой " **изолировать выделенный элемент** " есть два флажка: флажок **Показать все родители** и **сохранить только родителей с новым контекстом создания стека** .  

Флажок **Показать все родители** включен по умолчанию.  Чтобы отобразить на холсте элемент и все родительские элементы, выберите элемент и нажмите кнопку **изолировать выделенный элемент** .  

Чтобы отобразить на холсте элемент и родительский объект с новым контекстом размещения, включите параметр **сохранить только родители с помощью нового контекста** и нажмите кнопку **изолировать выделенный элемент** .  

Чтобы отобразить элемент, выбранный на холсте, отключите оба параметра и нажмите кнопку **изолировать выбранный элемент** .  

В нижней части области **3D DOM** найдите **элементы Hide с тем же порядком рисования, что и их родительский** элемент CheckBox.  При нажатии кнопки CheckBox вы обновляете элементы на основе вашего выбора.  Если выбрано, элементы, имеющие общий порядок рисования, будут сведены к родительскому элементу.  

Эти параметры предназначены для того, чтобы удалить некоторые сложные веб-страницы, которые будут создаваться на холсте.  

### Тип цвета «Z-индекс»  

Это различные зрительные образы, которые можно использовать для DOM на холсте.  Независимо от того, используется ли вы для экспериментов или потому что зрительные образы помогают лучше визуализировать модель DOM, DevTools имеет разные colorways и параметр " **использовать цвет фона** ".  На панели **Z-Index** в области **трехмерная модель DOM** развернуто **фиолетовые до белого** **цвета и цвет фона** .  С учетом добавленного визуального элемента меток z-index, отзыв, который привел к уменьшению числа вариантов цвета.  Новая простота упрощает отладку z-индексов.  Переключатели позволяют переключать параметры и выбирать тип цвета.  Тип цвета является либо наиболее подходящим для вашего проекта, либо тем, который вы хотели бы чаще всего.  

## ТРЕХМЕРНАЯ МОДЕЛЬ DOM  

:::image type="complex" source="../media/3d-view-dom-purple-box.msft.png" alt-text="Представление DOM" lightbox="../media/3d-view-dom-purple-box.msft.png":::
   Представление DOM  
:::image-end:::  

Если вы хотите использовать более общее представление отладки, а не z-index, **трехмерная модель DOM** обеспечивает общий вид модели DOM.  Поскольку контекст z-index удаляется, модель DOM более детально и четко задается.  Панель **3D DOM** имеет аналогичные функции, но есть несколько нюансов.  

### Изменение представления  

На панели **3D DOM** кнопка " **изолировать выделенный элемент** " **включает в себя дочерние элементы** и флажки " **включить родителей** ".  Оба флажка включены по умолчанию.  Это означает, что при нажатии кнопки **изолировать выбранный элемент** после выбора элемента холст отображает выбранный элемент, родительский элемент и его дочерние элементы.  Отключите параметр **Включить дочерние элементы** и снова нажмите кнопку **изолировать выбранный элемент** , чтобы отобразить выбранный элемент и родительский элемент.  Если включить параметр **Включить дочерние элементы** и отключить параметр **включить родители** , а затем нажать кнопку **изолировать выбранный элемент** , холст отобразит элемент и все дочерние элементы.  Если отключить оба параметра и нажать кнопку **изолировать выделенный элемент** , на холсте будет отображаться только тот элемент, который вы выбрали ранее.  

Ползунок на панели управления с именем **уровень вложенности для страницы** с числом рядом с ним.  Число показывает количество слоев документа.  Перетаскивание ползунка влево приводит к тому, что внешние слои будут отключаться до тех пор, пока не будет оставлено установленный уровень вложенности `1` , который отображает только элемент самой дальней стороны в DOM.  Чтобы устранить ненужные элементы, перетащите ползунок.  Это поможет лучше узнать, что происходит на нижних уровнях.  

### Тип цвета DOM  

На панели **3D DOM** отображаются следующие параметры.  

*   Три разных colorwaysа.  
    *   **Heatmap — фиолетовый и белый**  
    *   **Heatmap (синий) до желтого**  
    *   **Heatmap-Радуга**  
*   **Использование цвета фона**  
*   **Использование текстуры экрана**  
    
Параметр " **использовать текстуру экрана** " позволяет добавлять контекст в среду отладки.  Оно непосредственно отображает содержимое веб-страницы на элементы.

## Взаимодействие с командой средств разработчика Microsoft Edge  

Группа Microsoft Edge Devtools работает над пользовательским интерфейсом и добавляет дополнительные функции в 3D-представление на основе вашего отзыва.  Отправьте свой отзыв, чтобы помочь вам улучшить Microsoft Edge DevTools.  Щелкните значок **Отправить отзыв** в DevTools или выберите `Alt` + `Shift` + `I` в Windows или Linux или `Option` + `Shift` + `I` на macOS и введите любые запросы на отзыв или возможности, необходимые для DevTools.
