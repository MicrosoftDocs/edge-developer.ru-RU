# <a name="navigate-microsoft-edge-devtools-with-assistive-technology"></a>Навигация по Microsoft Edge DevTools с помощью вспомогательных технологий  

Следующая статья направлена на помощь пользователям, которые в основном полагаются на вспомогательные технологии, такие как доступ к читателям экрана и использование [Microsoft Edge DevTools][MicrosoftEdgeDevtoolsMain].  [Microsoft Edge DevTools —][MicrosoftEdgeDevtoolsMain] это набор средств веб-разработчика, встроенных в браузер Microsoft Edge.  Если вы ищете функции DevTools, связанные с улучшением доступности веб-страницы, перейдите к [ссылке доступности][DevtoolsAccessibilityReference].  

Доступность DevTools — это работа в процессе работы.  Некоторые панели и вкладки лучше работают с вспомогательными технологиями, чем другие.  В этом руководстве вы проходите по панелям, которые являются наиболее доступными, и выделяет определенные проблемы, с которыми вы можете столкнуться по пути.  

## <a name="overview"></a>Обзор  

Перед запуском помогает иметь умственную модель структуры пользовательского интерфейса DevTools.  DevTools делится на ряд панелей, организованных в [таблистер ARIA.][W3CWaiAriaTablist]  

Например:  

*   Средство **Elements** позволяет просматривать и изменять узлы DOM[DevtoolsDomIndexNavigateDomTreeKeyboard] или [CSS][DevtoolsCssIndex].  
*   Панель [консоли позволяет][DevtoolsConsoleIndex] читать журналы JavaScript и изменять объекты в режиме live.  

В области контента каждой панели существует ряд различных средств, часто именуемого вкладками или панелями в документации.  
Например, средство **Elements** содержит дополнительные вкладки для проверки слушателей событий, дерева доступности и многое другое.  Различие между вкладками и области несколько произвольно.  Единственная причина, по которой вы можете просмотреть один термин или другой, это сохранить согласованность с остальной частью официальной документации DevTools.  

## <a name="keyboard-shortcuts"></a>Сочетания клавиш  

Ссылка на ярлыки клавиатуры [DevTools Keyboard] [DevtoolsShortcuts] — это полезная таблица читов.  Обязательно закладки и вернуться к ней при изучении различных панелей.  

## <a name="open-devtools"></a>Открыть средства разработчика  

Чтобы начать работу, перейдите в [Open Microsoft Edge DevTools][DevtoolsOpen].  Существует несколько способов открытия DevTools, как с помощью клавиш, так и элементов меню.  

## <a name="navigate-between-panels"></a>Перемещение между панелями  

### <a name="navigate-by-keyboard"></a>Навигация по клавиатуре  

*   С открытием DevTools выберите `Control` + `]` \(Windows, Linux\) или `Command` + `]` \(macOS\) для фокуса следующей панели.  
*   Выберите `Control` + `[` \(Windows, Linux\) `Command` + `[` или \(macOS\) для фокуса предыдущей панели.  
*   Можно также использовать для перемещения фокуса в список ARIA панели и использования клавиш стрелки для изменения панелей, хотя это может быть быстрее, чтобы использовать ранее упомянутые `Shift` + `Tab` ярлыки. [][W3CWaiAriaTablist]  

**Известные проблемы**  

*   Некоторые панели, например средства **консоли** и **производительности,** могут перемещать фокус в область контента панели, как только каждая панель активируется.  Это может затруднить навигацию по клавишам стрелки.  
*   Имя выбранной панели объявлено, но только после прочтет фокусное содержимое на панели.  Это может сделать его очень легко пропустить.  

### <a name="navigate-by-command-menu"></a>Навигация по командным меню  

Чтобы сосредоточиться на определенной панели, используйте [командное меню:][DevtoolsCommandMenuIndex]  

1.  С открытием DevTools выберите `Control` + `Shift` + `P` \(Windows, Linux\) или `Command` + `Shift` + `P` \(macOS\) для открытия **командного меню.**  
    Командное **меню** — это нечеткий комбобокс поиска.  
1.  Введите имя панели, которая необходимо открыть, а затем используйте клавиатуру, чтобы перейти `Down Arrow` к правильному варианту.  
1.  Выберите `Enter` для запуска команды.  

Выполните следующие действия, чтобы открыть **средство Elements.**  

1.  Откройте **командное меню.**  
1.  Введите `E` затем `L` .  Выбран **> Элементов** show.  
1.  Выберите `Enter` команду, открываемую панель.  

Откройте панель таким образом, чтобы фокусироваться на содержимом панели.  В случае с инструментом **Elements** фокус перемещается в **дерево DOM.**  

## <a name="elements-panel"></a>Панель Elements  

### <a name="inspect-an-element-on-the-page"></a>Проверка элемента на странице  

1.  Перейдите к элементу, который необходимо проверить с помощью курсора в считывателье экрана.  
1.  Имитация правой кнопкой мыши на элементе для открытия контекстного меню.  
1.  Выберите параметр **Inspect.**  Это [открывает панель Elements и фокусирует элемент в dom Tree][DevtoolsDomIndexViewDomNodes].  

Дерево **DOM выложено** как [дерево ARIA.][W3CWaiAriaTree]  Например, перейдите по ссылке [Перейдите по **dom Tree** с помощью клавиатуры][DevtoolsDomIndexNavigateDomTreeKeyboard].  

### <a name="copy-the-code-for-an-element-in-the-dom-tree"></a>Скопируйте код элемента в дереве DOM  

1.  Сосредоточив внимание на узле **в дереве DOM,** наведите курсор на узел и откройте контекстное меню \(правой кнопкой мыши\).  
1.  Расширь **параметр Copy.**  
1.  Выберите **скопируйте внешнийHTML**.  

**Известные проблемы**  

*   **Копирование внешнегоhtML** часто не выбирает текущий узел, а вместо этого выбирает родительский узел.  Однако содержимое элемента по-прежнему должно быть в скопированной внешней частиHTML.  

### <a name="modify-the-attributes-of-an-element-in-the-dom-tree"></a>Изменение атрибутов элемента в дереве DOM  

*   С упором на узел в **dom Tree**выберите, `Enter` чтобы сделать его редактирования.  
*   Выберите `Tab` для перемещения между значениями атрибута.  Когда вы слышите "пространство", вы находитесь внутри пустого ввода текста и можете ввести новое значение атрибута.  
*   Выберите `Control` + `Enter` \(Windows, Linux\) `Command` + `Enter` или \(macOS\), чтобы принять изменение и услышать все содержимое элемента.  

**Известные проблемы**  

*   При вводимом текстовом вводе вы не получите никакой обратной связи.  Если вы делаете опечатки и используете клавиши стрелки для изучения ввода, вы также не получите никакой обратной связи.  Самый простой способ проверить работу — принять изменение, а затем прослушать весь элемент, который будет объявлен.  

### <a name="edit-the-html-of-an-element-in-the-dom-tree"></a>Изменение HTML элемента в дереве DOM  

*   С упором на узел в **dom Tree**выберите, `Enter` чтобы сделать его редактирования.  
*   Выберите `Tab` для перемещения между значениями атрибута.  Например, когда вы услышите имя элемента, вы находитесь внутри ввода текста и можете изменить `h2` тип элемента.  
*   Чтобы принять изменение, выберите `Control` + `Enter` \(Windows, Linux\) `Command` + `Enter` или \(macOS\).  

Например, при введите и выберите `h3` `Control` + `Enter` \(Windows, Linux\) или `Command` + `Enter` \(macOS\), `h3` начните и выберите конечные теги элемента.  

## <a name="elements-tool-panels"></a>Панели инструментов Elements  

Средство **Elements** содержит дополнительные вкладки для проверки вещей, например CSS, примененных к элементу или соответствующему месту в дереве доступности.  

*   С упором на узел в **dom Tree**выберите, пока не услышите, что выбрана `Tab` области стилей. ****  
*   Используйте эту `Right Arrow` таблицу для изучения других доступных вкладок.  

Дерево **DOM превращает** элементы с атрибутами в фокусируемые ссылки, поэтому для достижения области `href` `Tab` **Стилей** может потребоваться выбрать несколько раз.  

**Известные проблемы**  

Вкладки **doM Breakpoints** и **Properties** недоступны для клавиатуры.  

### <a name="styles-pane"></a>Области стилей  

В области **Стилей** найдите элементы управления для стилей фильтрации, чтобы окантовать состояния элементов \(например:active и [:focus\),][MDNFocus]переувязывание классов и добавление новых классов. [][MDNActive]  Существует также мощный инструмент проверки стиля для изучения и изменения стилей, применяемых в настоящее время к элементу, который находится в центре **внимания в dom Tree**.  

Ключевое понятие, которое необходимо понять в области **Стилей,** заключается в том, что в ней показаны только стили для выбранного в настоящее время узла **в дереве DOM.**  Например, предположим, что вы закончили проверку стилей узла, и теперь необходимо взглянуть на стили `<header>` `<footer>` для узла.  Для этого сначала необходимо выбрать `<footer>` узел **в дереве DOM.**  Вы можете быстрее использовать рабочий процесс [Inspect](#inspect-an-element-on-the-page) для проверки узла, который находится в общей близости от узла \(например, ссылки в подножке\), которая фокусирует `footer` дерево **DOM,** а затем с помощью клавиатуры перейдите к точному узлу, в котором вы заинтересованы.  

#### <a name="navigate-the-styles-pane"></a>Навигация по области Стилей  

Поскольку все средства стиля так или иначе подключаются к области **Стилей,** имеет смысл сначала стать экспертом в этом инструменте.  

*   Сосредоточив внимание на области **Стилей,** выберите для `Tab` перемещения фокуса внутрь и изучения содержимого.  
*   Выберите, `Tab` пока первый стиль не станет активным.  Если вы используете считыватель экрана, этот первый стиль объявлен как `element.style {}` .  
*   Выберите `Down Arrow` для перемещения по списку стилей в порядке специфичности.  Считыватель экрана объявляет каждый стиль, начиная с имени CSS-файла, номера строки, на котором отображается стиль, и имени стиля.  Например, `main.css:233 .card__img {}`.  
*   Выберите `Enter` для более подробной проверки стиля.  Фокус начинается с редактируемой версии имени стиля.  
*   Выберите `Tab` для перемещения между редактируемыми версиями каждого свойства CSS и соответствующими значениями.  В конце каждого блока стилей находится пустое текстовое поле, которое можно использовать для добавления дополнительных свойств CSS.  
*   Вы можете продолжить выбор для перемещения по списку стилей или для выхода из режима и обратного перемещения `Tab` `Escape` по клавишам стрелки.  

Дополнительные ярлыки перейдите к ссылке [Стили клавиатуры области][DevtoolsShortcutsStylesPaneKeyboard].  

**Известные проблемы**  

*   Если вы используете **текстовое** поле Для редактирования фильтра, вы больше не сможете перемещаться по списку стилей.  

#### <a name="toggle-element-state"></a>Состояние элемента Toggle  

Чтобы скомплектовка состояния элемента, например `:active` или `:focus` :  

1.  Перейдите к области **Стилей** и `Tab` выберите, пока кнопка Состояние элемента **Toggle** не будет фокусироваться.  
1.  Выберите, `Enter` чтобы расширить коллекцию состояния элементов.  Состояния элементов представлены в качестве группы почтовых ящиков.  
1.  Выберите `Tab` до первого состояния, имеет `:active` фокус.  
1.  Выберите, `Space` чтобы включить его.  Если выбранный в настоящее время элемент dom Tree имеет стиль, он `:active` теперь применяется.  
1.  `Tab`Удерживайте, чтобы изучить все доступные состояния.  

#### <a name="add-an-existing-class"></a>Добавление существующего класса  

Рядом с **кнопкой Состояние элемента Toggle** находится **кнопка Element Classes.**  Чтобы переместить фокус на него, выберите `Tab` и выберите `Enter` .  Фокус перемещается в поле редактирования текста с меткой **Add New Class**.  

Кнопка **Element Classes** используется главным образом для добавления существующих классов в элемент.  Например, если в таблице стилей содержится класс помощника с именем, вы можете выбрать внутри области редактирования текста, чтобы отобразить список предложений классов и использовать его для поиска `.clearfix` `.` `Down Arrow` `.clearfix` предложения.  Или введите имя класса самостоятельно и выберите `Enter` его применение.  

#### <a name="add-a-new-style-rule"></a>Добавление нового правила стиля  

Рядом с **кнопкой "Классы элементов"** находится **кнопка "Правило нового** стиля".  Чтобы переместить фокус на него, выберите `Tab` и выберите `Enter` .  Фокус перемещается в редактируемое текстовое поле внутри инспектора стиля.  Исходное текстовое содержимое поля — это имя тега элемента, выбранного в **дереве DOM.**  
В этом поле можно ввести любое имя класса, а затем назначить ему свойства `Tab` CSS.  

### <a name="computed-tab"></a>Вычисляемая вкладка  

Сосредоточив внимание на **вкладке Computed,** выберите `Tab` для перемещения фокуса внутрь и изучения содержимого.  В **вкладке Computed** есть элементы управления для изучения того, какие свойства CSS фактически применяются к элементу в порядке определенности.

#### <a name="explore-all-computed-styles"></a>Изучение всех вычислительных стилей  

Выберите, `Tab` пока не достигнете коллекции вычислительных стилей.  Они представлены в качестве [дерева ARIA][W3CWaiAriaTree].  Расширение списка показывает, какие селекторы CSS применяют вычислительный стиль.  Эти селекторы организованы по определенности.  Считыватель экрана объявляет вычислительные значения, которые в настоящее время совпадает с селектором CSS, имя файла таблицы стилей, которая содержит селектор, и номер строки для селектора.  

**Известные проблемы**  

*   Если вы используете текстовое поле **Filter,** вы больше не сможете проверять стили.  

### <a name="event-listeners-tab"></a>Вкладка слушателей событий  

В **инструменте Elements** вы можете проверить слушателей событий, примененных к элементу с помощью вкладки **"Слушатели** событий".  С упором на **панель Стилей** выберите переход к панели `Right Arrow` **слушателей** событий.  

#### <a name="explore-event-listeners"></a>Изучение слушателей событий  

Слушатели событий представлены в качестве [дерева ARIA][W3CWaiAriaTree].  Для навигации по ним можно использовать клавиши стрелки.  Считыватель экрана объявляет имя объекта DOM, к котором подключен прослушиватель событий, а также имя файла, в котором определен прослушиватель событий, и номер строки.  

### <a name="accessibility-pane"></a>Области доступности  

С упором на **области доступности** выберите переместить фокус `Tab` внутрь и изучить содержимое.  На области [доступности][DevtoolsAccessibilityReference] есть элементы управления для изучения дерева доступности, атрибуты ARIA, применяемые к элементу, и свойства вычислительной доступности.  

#### <a name="accessibility-tree"></a>Дерево доступности  

Дерево **доступности** представлено в качестве [дерева ARIA,][W3CWaiAriaTree] где каждый соответствует элементу `treeitem` DOM.  Дерево объявляет вычисляемую роль выбранного узла.  Общие элементы, `div` как `span` и объявляются как "GenericContainer" в дереве.  Используйте клавиши стрелки для обхода дерева и изучения отношений между родителями и детьми.  

**Известные проблемы**  

*   Тип дерева [ARIA,][W3CWaiAriaTree] используемого в области **доступности,** может быть неправильно выставлен в Microsoft Edge для считывателей экрана macOS, таких как VoiceOver.  Подписаться [на проблему Chromium #868480][ChromiumIssues868480] быть информированы о ходе работы по этому вопросу.  
*   Каждый из разделов **Атрибуты ARIA** и **Computed Properties** помечен как [дерево ARIA,][W3CWaiAriaTree]но в настоящее время каждый из них не имеет управления фокусом и не является клавиатурным операблемым.  

## <a name="audits-panel"></a>Панель аудитов  

Инструмент **аудита** должен выполнить серию тестов на сайте для проверки распространенных проблем, связанных с производительностью, доступностью, SEO и рядом других категорий.  

### <a name="configure-and-run-an-audit"></a>Настройка и запуск аудита  

1.  При первом **запуске** средства аудита фокусируется на кнопке **Выполнить** аудит в конце формы.  По умолчанию форма настроена для проведения аудитов для каждой категории с помощью мобильной эмуляции на смоделированном 3G-соединении.  
1.  Чтобы изменить параметры аудита, используйте или перейдите обратно в `Shift` + `Tab` режиме Обзор.  
1.  Когда вы будете готовы к запуску аудита, перейдите к кнопке **Выполнить аудит** и выберите `Enter` .  
1.  Фокус перемещается в модальный окне с кнопкой **Отмена,** которая позволяет выйти из аудита.  Вы можете услышать серию наушников во время аудита и обновления страницы несколько раз.  

**Известные проблемы**  

*   Различные разделы формы конфигурации в настоящее время не помечены `fieldset` элементом.  Может быть проще перемещаться по ним в режиме Обзор, чтобы выяснить, какие элементы управления связаны с каждым разделом.  
*   По завершению аудита не существует объявления earcon или live region.  Обычно это занимает около 30 секунд, после чего вы сможете перейти к результатам.  Использование режима Просмотр может быть самым простым способом достижения результатов.  

### <a name="navigate-the-audit-report"></a>Навигация по отчету о аудите  

Отчет аудита организован в разделы, соответствующие каждой из категорий аудита.  Отчет открывается со списком баллов для каждой категории.  Эти оценки также являются ссылками, которые можно использовать для перехода в соответствующие разделы.  В каждом разделе находятся расширяемые элементы, содержащие сведения, относящиеся к пройденным или неудачным `details` аудитам.  По умолчанию показаны только сбои аудита.  Каждый раздел заканчивается `details` заключительный элемент, содержащий все пройденные аудиты.  

Чтобы выполнить новый аудит, используйте `Shift` + `Tab` для выхода из отчета и найдите кнопку **Выполнить аудит.**
