# Коды ошибок и состояний Console  

Этот справочник поможет вам разобраться в сообщениях об ошибках и состояниях инструментов разработчика [Console](../console.md).  Код нужно искать в приведенной ниже таблице по его префиксу \(где "x" обозначает 0&ndash;9-значный номер\):

| Префикс кода | Область | Описание |  
|:--- | :--- |:--- |  
| [CSP143xx](#csp-codes) | Политика в отношении безопасности содержимого | Ресурсы, блокируемые с помощью HTTP-заголовка `Content-Security-Policy`.  |  
| [CSS31xx](#css-codes) | Таблицы стилей | Ошибки, связанные с проблемами источников и узловых серверов шрифтов *Web Open Font Format* \(WOFF\) и *Embedded OpenType* \(EOT\).  |  
| [HTML1112–1300](#html-codes) | HTML | Чаще всего это устаревшие коды, относящиеся к таким понятиям Microsoft Internet Explorer, как *режим документа* и *просмотр в режиме совместимости*.  |  
| [HTML1400–1532](#html5-parser-warnings) | Анализ HTML5 | Предупреждения проверки, создаваемые средством анализа HTML5.  |  
| [HTTP](#http-codes) | Ошибки и состояние сервера | Коды, возвращаемые удаленными серверами в ответ на HTTP-запросы.  |  
| [SCRIPT10xx](#javascript-syntax-errors) | Ошибки синтаксиса JavaScript | Возникают, когда структура одной из инструкций JavaScript нарушает одно или несколько синтаксических правил.  |  
| [SCRIPT50xx](#javascript-run-time-errors) | Ошибки выполнения JavaScript | Возникают, когда сценарий пытается выполнить действие, которое система выполнить не может.  |  
| [SEC71xx](#security-codes) | Безопасность | Отображает условия безопасности, выполнения которых требует Microsoft Edge, такие как *смешанное содержимое* и *защита от отслеживания*.  |  
| [SVG560x](#svg-codes) | Масштабируемая векторная графика |  Расширенная отладка SVG в настоящее время не поддерживается, но некоторые сообщения Console предоставляются в целях отладки.  |  
| WebGL11_xxx | WebGL | Сообщения об ошибках из контекста WebGL.  См. также [Ошибки GLSL](https://msdn.microsoft.com/library/dn611835.aspx).  |  
| [XML5xxx](#xml-codes) | XML | Ошибки анализа и проверки XML.  |  

## Коды CSP  

Отчеты о ресурсах, блокируемых с помощью HTTP-заголовка `Content-Security-Policy`, представляются через средство разработчика Console и при необходимости возвращаются на сервер.

| Код | Сообщение | Описание | Предлагаемое исправление |
|:--- |:--- |:--- |:--- |
| CSP14301 | "Не удалось выполнить анализ политики \[тип политики\], так как \<the reason for canceling the operation\>. Политика будет проигнорирована". | Не удалось выполнить анализ политики безопасности указанного типа \(например, script-src, base-uri\) по упомянутой причине, и политика будет проигнорирована.  | Убедитесь в том, что перечислили все необходимые ресурсы определенного типа в одной директиве.  Например, в `script-src https://host1.com; script-src https://host2.com` вторая директива будет проигнорирована.  В следующей записи оба происхождения указаны верно: `script-src https://host1.com https://host2.com`.  |
| CSP14302 | "Не удалось выполнить анализ источника в \[тип политики\] для директивы \[тип директивы\] по адресу \[URL-адрес источника\]. Источник будет проигнорирован".                                                         | Для большинства директив CSP требуется один или несколько источников содержимого \(URL-адресов для скачивания содержимого\).  Эта ошибка указывает на политику с директивой, содержащей URL-адрес источника, который не удалось проанализировать.  | Проверьте источник содержимого \(чаще всего это URL-адрес\), определенный в указанной директиве, которую можно найти в политике упомянутого типа.  Исправьте или замените URL-адрес источника либо удалите директиву.  <br /> *\*Ваша политика должна содержать директиву политики default-src в качестве резервного варианта для ресурсов других типов, не имеющих собственной политики.* |
| CSP14303 | "Политика [тип политики] была пуста". | Политика указанного типа \(например, Content-Security-Policy в HTTP-заголовке\) пуста.  | Краткий справочник по настройке политики в отношении безопасности содержимого можно найти в статье [http://content-security-policy.com](http://content-security-policy.com).  |
| CSP14304 | "Неизвестный источник \[URL-адрес источника\] для директивы \[тип директивы\] в источнике политики \[тип политики\] будет проигнорирован". | Источники утвержденного \(безопасного\) содержимого в указанной директиве неизвестны и будут проигнорированы.  | Проверьте, правильно ли указан источник содержимого \(часто это URL-адрес\).  |
| CSP14305 | "Неподдерживаемый источник \[URL-адрес источника\] для директивы[тип директивы] в источнике политики \[тип политики\] будет проигнорирован". | Источник \(URL-адрес, ключевое слово или данные), указанные в этой директиве, не поддерживается и будет проигнорирован.  | Адрес сайта-источника может включать необязательный подстановочный знак \ (звездочку, `*`\).  Например, http://`*`.foo.com или mail.foo.com:*.  Узлы разделяются пробелами.  Источниками могут быть также ключевые слова \(все перечисленное поддерживается: none, self, unsafe-inline и unsafe-eval\) или данные \(data:URIs, mediastream:URIs\).  |
| CSP14306 | "Не заданы источники для директивы \[тип директивы\] \[тип политики\]. Это эквивалентно использованию ключевого слова none, поэтому никакие ресурсы этого типа скачиваться не будут". | Указание директивы без источников безопасного содержимого — это то же самое, что отсутствие разрешения на скачивание любых ресурсов, относящихся к указанному в директиве типу.  | Источник может быть одним или несколькими именами Интернет-узлов или IP-адресами, а также (необязательно) схемой URL-адресов и/или номером порта.  |
| CSP14307 | "Источник [URL-адрес источника]уже упомянут в директиве \[тип директивы\] для \[тип политики\]". | В этой директиве повторно указан источник \(URL-адрес, ключевое слово или данные). Он будет проигнорирован.  | Удалите повторное указание источника.  |
| CSP14308 | "Не удалось проанализировать директиву для \[тип политики\] в [имя директивы]". | Анализ указанной директивы не дал результатов.  Рекомендации по написанию поддерживаемых директив см. в статье [http://content-security-policy.com](http://content-security-policy.com/).  | Директивы должны быть разделены точкой с запятой.  Например, если у вас есть приложение, которое загружает все свои ресурсы из сети доставки содержимого \(например, `https://cdn.example.net`\), и вы знаете, что вам не нужно ни содержимое фреймов, ни подключаемые модули, то ваша политика может выглядеть, например, так: `Content-Security-Policy: default-src https://cdn.example.net; child-src 'none'; object-src 'none'.` *\*Хотя директивы разделяются точкой с запятой, источники внутри директивы должны разделяться только пробелом.* |
| CSP14309 | "Неизвестная директива \[имя директивы\] в \[тип политики\]. Директива будет проигнорирована". | Директива, указанная в политике CSP, неизвестна и будет проигнорирована.  | Список поддерживаемых директив см. в разделе [http://content-security-policy.com](http://content-security-policy.com/).  |
| CSP14310 | "Неподдерживаемая директива [имя директивы] в \[тип политики]. Директива будет проигнорирована". | В ходе анализа в типе политики \(например, в поле заголовка `Content-Security-policy-Report-Only`\) была найдена неподдерживаемая директива, которая будет проигнорирована.  Список поддерживаемых директив см. в разделе [http://content-security-policy.com](http://content-security-policy.com/).  | Удалите неподдерживаемую директиву.  **ПРИМЕЧАНИЕ**. Некоторые директивы не поддерживаются в элементе \<meta\> и в поле заголовка `Content-Security-policy-Report-Only`.  |
| CSP14311 | "Директива \[имя директивы\] уже указана в \[тип политики\]. Повторяющаяся директива будет проигнорирована". | В процессе анализа была обнаружена повторяющаяся директива. Вторая директива и относящиеся к ней указания источников будет проигнорирована.  | Удалите повторяющийся сценарий.  |
| CSP14312 | "Ресурс нарушил директиву[имя директивы] в [тип политики]: [целевой URI].  Ресурс будет заблокирован". | В этом примере: "Ресурс нарушил директиву 'script-src ms-appx: data: 'unsafe-eval' в политике Host Defined Policy: inline script.  Ресурс будет заблокирован". | Встроенный сценарий \(целевой код uri\) был заблокирован из-за директивы `'script-src ms-appx: data: 'unsafe-eval'` в политике host defined.  <br /> Авторам следует сделать все встроенные сценарии и стили внешними, поскольку агент пользователя не может определить, не был ли встроенный сценарий добавлен злоумышленником.  Удалите встроенный сценарий и разместите его во внешнем файле.  |
| CSP14313 | "Ресурс нарушил директиву[имя директивы] в [тип политики]: [целевой URI].  Ресурс не будет заблокирован, так как действует политика «только отчет»". | Был выявлен ресурс, нарушающий директиву, указанную в поле заголовка `Content-Security-policy-Report-Only`.  Поскольку ресурс относится к политике типа `Report-Only`, он не будет заблокирован.  | Если необходимо заблокировать этот ресурс в целях защиты сайта, переместите директиву в поле заголовка Content-Security-Policy.  |
| CSP14314 | "Не удалось отправить отчет о нарушении политики в [URI назначения], так как [причина отмены операции]". | Возникла проблема с отчетом о нарушении политики или с целевым расположением для отправки отчета. Должна быть указана причина, по которой отчет не был отправлен.  | Директива `report-uri` задает URL-адрес, по которому браузер должен отправлять отчеты при нарушении политики безопасности содержимого.  *\*Использование в тегах \<meta\> не предусмотрено.* |
| CSP14315 | "Не удалось применить директиву песочницы в [тип политики], поскольку [причина отмены операции]". | Сбой применения директивы песочницы по указанной причине.  Эта директива накладывает ограничения на действия, совершаемые страницей, а не на ресурсы, которые она может загружать.  Если имеется директива песочницы, то страница рассматривается как загруженная во внутреннем фрейме.  Таким образом можно заблокировать появление всплывающих окон, запуск подключаемых модулей и выполнение сценариев.  | Оставьте значение директивы песочницы пустым, чтобы сохранить все ограничения, или добавьте значения: `allow-forms`, `allow-same-origin`, `allow-scripts` и `allow-top-navigation`.  *\*Директива песочницы не поддерживается в элементе \<meta\> и в поле заголовка `Content-Security-policy-Report-Only`.* |
| CSP14316 | "Оценка сценария разрешена из-за переопределения узла". | При включении директивы `script-src` или`default-src` встроенный сценарий и `eval()` отключаются, если вы не укажете `unsafe-inline` и `unsafe-eval` соответственно.  | `'unsafe-eval'` позволяет использовать `eval()` и аналогичные методы для создания кода из строк.  Необходимо использовать одинарные кавычки.  ***ПРИМЕЧАНИЕ**. Это небезопасно, поскольку вы можете создать для своего веб-сайта уязвимости, связанные с межсайтовыми сценариями.* |
| CSP14317 | "Фрейм [имя источника] разрешен из-за переопределения узла". | Указанный фрейм-источник разрешен из-за переопределения политики CSP узла.  | Политика может содержать выражение nonce-source, означающее, что источник можно использовать только один раз и что сервер должен создавать новое значение для директивы при каждой передаче политики.  Такие выражения переопределяют ограничения в тех директивах, в которых они содержатся.  |

> [!NOTE]
> Microsoft Edge не проверяет тип MIME таблицы стилей для веб-сайтов в доверенной пользовательской зоне безопасности.  

## Коды CSS  

Эти ошибки имеют формат CSS31xx и связаны с проблемами источника и узлового сервера шрифтов *Web Open Font Format* \(WOFF\) и *Embedded OpenType* \(EOT\).  

| Код | Сообщение | Описание | Предлагаемое исправление |
|:--- |:--- |:--- |:--- |
| CSS3111 | "Правило @font-face обнаружило неизвестную ошибку" | Возникла неизвестная проблема, связанная с шрифтом WOFF или EOT таблицы CSS.  | Проверьте источник шрифтов WOFF.  Попробуйте выбрать другое начертание шрифта или другой источник, чтобы узнать, можно ли воспроизвести проблему.  |
| CSS3112 | "Правилу @font-face не удалось выполнить проверку целостности WOFF" | Возможно, шрифт WOFF поврежден, неполон или имеет неправильный формат.  | Проверьте источники шрифтов.  Попробуйте выбрать известный шрифт или источник, чтобы узнать, можно ли воспроизвести проблему.  |
| CSS3113 | "Правило @font-face обнаружило несоответствие между происхождением документа и корневой строкой EOT" | Шрифт нельзя использовать, поскольку URL-адрес \(корневая строка\) шрифта EOT не соответствует домену документа, в котором используется этот шрифт.  | Возможно, неверна контрольная сумма URL-адреса в корневой строке шрифта EOT, что указывает на повреждение или изменение URL-адреса шрифта.  Убедитесь в том, что этот шрифт лицензирован или что он разрешен для веб-сайта, на котором он используется.  |
| CSS3114 | "Правилу @font-face не удалось проверить разрешения на внедрение шрифтов OpenType.  Разрешение должно иметь значение Installable". | У шрифта нет разрешений на установку для текущей веб-страницы.  | Получите нужное разрешение или лицензии для внедрения шрифта.  |
| CSS3115 | "Правилу @font-face не удалось загрузить недопустимый шрифт OpenType". | В данном случае этот шрифт использовать нельзя.  | Получите разрешение или лицензии для действительного шрифта.  |
| CSS3116 | "Правилу @font-face не удалось выполнить запрос о перекрестном происхождении.  Нет заголовка No Access-Control-Allow-Origin". | Возможно, шрифт не настроен для междоменного доступа.  | Шрифт не обслуживается из того же источника, что и документ.  Убедитесь в наличии разрешения от узла, обслуживающего шрифт, на использование этого шрифта с помощью HTTP-заголовка Access-Control-Allow-Origin.                        |
| CSS3117 | "Правилу @font-face не удалось выполнить запрос о перекрестном происхождении.  Доступ к ресурсу ограничен". | Заголовок Access-Control-Allow-Origin может быть неправильно настроен, или узел шрифта может не разрешать использовать этот шрифт на вашей странице.  | Убедитесь в том, что у ресурса есть соответствующие разрешения, а также правильно настроенный HTTP-заголовок отклика, в котором указано происхождение для междоменного доступа, на обслуживающем шрифты узле.  |
| CSS3118 | "Не удалось создать новую таблицу стилей.  В документе больше \[максимальное значение\] таблиц стилей". | В процессе отрисовки вашей страницы Microsoft Edge создал более 4095 объектов типа "таблиц стилей".  | Это может быть вызвано бесконтрольным процессом JavaScript или системной ошибкой сборки.  Сократите количество создаваемых объектов типа "таблица стилей".  |
| CSS3119 | "Запрос мультимедиа -ms-view-state устарел.  Запросы мультимедиа -ms-view-state могут быть изменены или отключены в выпусках после Windows 8.1.  Вместо них используйте запросы max-width и min-width." | Ваша таблица CSS содержит запрос мультимедиа `-ms-view-state`.  | Используйте max-width и min-width.  |

## HTML-коды

Эти коды имеют формат HTML1xxx, например HTML1115.  Часто это устаревшие коды, относящиеся к таким понятиям Internet Explorer, как *режим документа* и *просмотр в режиме совместимости*.  

| Код | Сообщение | Описание | Предлагаемое исправление |  
| :--- |:--- |:--- |:--- |  
| HTML1112 | "Перезапуск кодовой страницы с \[кодирование\] в \[кодирование\]" | Указана кодовая страница, отличающаяся от кодовой страницы сервера.  | Чтобы избежать этой ошибки, используйте ту же кодовую страницу, что и сервер.  |  
| HTML1113 | "Перезагрузка режима документов из \[режим\]" в \[режим\]" | Для веб-страницы требуется другой режим документов, отличный от того, который в настоящее время установлен в браузере.  | Это сообщение может появиться, если пользователь просматривает страницу с другими параметрами, что выводит задачу за пределы контроля разработчика.  |  
| HTML1114 | "Кодовая страница \[кодирование\] из \[домен\] переопределяет конфликтующую кодовую страницу \[кодирование\] из \[домен\]" | Кодовые страницы, указанные в заголовках HTTP \(с сервера\) и HTML \(внутри страницы\), различаются.  | Измените одну из них, чтобы они совпали.  |
