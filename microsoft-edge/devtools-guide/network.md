# Сеть

С помощью панели **Network (сеть** ) можно отслеживать, проверять и профилировать запросы и ответы, отправляемые по каналу связи. С его помощью вы можете:

 - [Просмотр записи всех запросов ресурсов](#network-summary) , сделанных страницей
 - [Измерение времени загрузки сайта](#summary-bar) для нового и возвращающего пользователей 
 - [Проверка заголовков, основных текстов сообщений, параметров и файлов cookie, которые](#request-details) обмениваются между страницей и сетью
 - [Определение сетевых событий, вызывающих узкие места](#timings) во время загрузки сайта

![Панель DevTools сети Microsoft Edge](./media/network.png)

## Сводка по сети

Когда вы открываете DevTools, сетевое Профилирование включается по умолчанию. Весь сетевой трафик с активной вкладки браузера записывается в списке "Сводка сети" даже в том случае, если вы работаете на другой DevTools панели, чем в *сети*.

![Индикатор выполняющегося профилирования в сети](./media/network_profile_indicator.png)

### панель инструментов;

На панели инструментов находятся элементы управления для профилирования и фильтрации активности сети на странице. 

![Панель инструментов "профилировщик сети"](./media/network_toolbar.png)

1. **Запуск и остановка сеанса профилирования**: по умолчанию используется сетевое профилирование, а сетевой трафик заносится в список [**сетевых профайлеров**](#network-request-list) . Вы можете отключить захват сети с помощью кнопки **Stop** "остановить `Ctrl+E` " ().

2. **Экспортировать как HAR**: вы можете сохранить текущий сеанс профилирования в сети ( `Ctrl+S` ) в виде файла [архива НТТР (HAR)](https://dvcs.w3.org/hg/webperf/raw-file/tip/specs/HAR/Overview.html) в формате JSON. 

3. **Фильтр по типу контента**: отфильтровать список сетевых запросов по конкретным запросам содержимого (*документы, таблицы стилей, изображения, сценарии, мультимедиа, шрифты, XHR, другие*). По умолчанию отображаются все типы контента.

4. **Find**: Filter ( `Ctrl+F` ) список сетевых запросов с именами записей (путями ресурсов), содержащими указанную строку поиска.

5. **Всегда обновлять с сервера**: нажатие этой кнопки приведет к принудительной загрузке ресурсов страницы из сети, а не кэша браузера. Вы можете обновить страницу по сети в один раз, нажав `Ctrl+F5` .

6. **Обход рабочего процесса для всех сетевых запросов**: отключение зарегистрированных сотрудников службы в качестве сетевых прокси. 

7. Кнопки "очистить"

   - **Очистить кэш**: удаляет все ресурсы, хранящиеся в кэше браузера (и эмулирует первый процесс загрузки страницы).
   - **Очистка файлов cookie**: удаляет все cookie-файлы для данного домена (и эмулирует процесс первого запуска сайта).
   - **Очистить записи на**навигации: записанный трафик очищается при навигации по страницам. Этот режим включен по умолчанию.
   - **Очистка сеанса**: удаляет все записи сетевого запроса из сводного списка **сети** .

### Список сетевых запросов

Весь сетевой трафик записывается в список (до тех пор, пока они не будут очищены при навигации, не будут удалены вручную или DevTools закрываются). Если щелкнуть любую запись, откроется более [подробное представление запроса](#request-details).

![Список сетевых запросов](./media/network_request_list.png)

Список сетевых запросов содержит следующие данные: 

Столбец | Описание 
:------------ | :------------- 
**Name** | Имя и URL-путь запроса
**Протокол** |  Тип протокола для запроса (например *, HTTPS, HTTP/2*)
**Метод** |    [Метод HTTP](https://developer.mozilla.org/docs/Web/HTTP/Methods) , используемый для запроса
**Результат** |    Код [состояния ответа HTTP](https://developer.mozilla.org/docs/Web/HTTP/Status)
**Тип содержимого** |  Тип запрашиваемого мультимедиа ([тип MIME](https://en.wikipedia.org/wiki/Media_type))
**Получено** | Размер ответа, доставленный сервером (не подсчитанный для кэшированных ответов)
**Время** |  Время для загрузки ответа сервера (не рассчитано для кэшированных ответов)
**Владельцы** | Подсистема, отвечающая за инициирование запроса (например *, средство синтаксического анализа, перенаправление, сценарий и т*. д.)
**Информация о сроках** | Визуальная временная шкала для сетевых событий запроса (например *, остановлена, разрешение (DNS), подключение (TCP), SSL, отправка, ожидание (TTFB), Загрузка*. При наведении указателя мыши на диаграмму выводятся более детальные сведения о [времени](#timings)сетевого соединения.

### Строка сводки

В нижней части панели " **сеть** " выводятся общие сведения об ошибках сети HTTP, запросах, передаче данных и времени загрузки во время сеанса профилирования в сети (например, так как DevTools открыл и записывал сетевой трафик).

![Строка сводки по сети](./media/network_summary_bar.png)

**Затраченное время** указывает время между началом сеанса профилирования и моментом, когда последний ресурс был загружен из сети. Ресурсы, извлекаемые из кэша браузера, не начисляются время на этот номер. 

**Время загрузки DOM** указывает время между началом сеанса профилирования и событием [DOMContentLoaded](https://developer.mozilla.org/docs/Web/Events/DOMContentLoaded) , чтобы указать, что структура документа страницы была загружена и проанализирована (хотя и не обязательно какие-либо таблицы стилей, изображения или подкадры).

Время **загрузки страницы** — это время между началом сеанса профилирования и событием [загрузки](https://developer.mozilla.org/docs/Web/Events/load) , чтобы показать, что документ страницы (и все его ресурсы) был полностью загружен.

## Сведения о запросе

Если щелкнуть любой элемент в списке [**Сводка сети**](#network-summary) , откроется область [**сведения о запросе**](#request-details) с дополнительными сведениями на каждой из указанных ниже вкладок.

![Область сведений о сетевом запросе](./media/network_request_details.png)

### Заголовки
Показывает [заголовки HTTP](https://developer.mozilla.org/docs/Web/HTTP/Headers) , отправляемые на сервер и получаемые от него. Щелкните правой кнопкой мыши любой элемент заголовка, чтобы скопировать его `Ctrl+C` в буфер обмена. Вы также можете выбрать несколько элементов, удерживая нажатой `Shift` клавишу или выделить все ( `Ctrl+A` ).

### Body
Отображает данные основного текста (при их наличии) в полезных данных запроса и ответа.

Содержимое изображения отображается с помощью измерений и размера данных.

Текстовое содержимое отображается в редакторе (только для чтения) с параметрами для форматирования minified содержимого с помощью **качественной печати** и (или) **переноса слов** для удобства чтения.

![Вкладка "текст" в области "сведения о запросе"](./media/network_details_body.png)

### Параметры
Выводит параметры строки запроса на получение запросов GET. Несмотря на то что параметры запросов POST отправляются в заголовках, запросы GET должны содержать их в URL-адресе. Они разбиваются на эту ссылку для упрощения чтения.

Щелкните правой кнопкой мыши любую строку, чтобы скопировать ее `Ctrl+C` в буфер обмена. Вы также можете выбрать несколько элементов, удерживая нажатой `Shift` клавишу или выделить все ( `Ctrl+A` ).

### Файлы cookie
Отображение файлов cookie, которые отправляются и принимаются в виде пар "ключ-значение".

Щелкните правой кнопкой мыши любую строку, чтобы скопировать ее `Ctrl+C` в буфер обмена. Вы также можете выбрать несколько элементов, удерживая нажатой `Shift` клавишу или выделить все ( `Ctrl+A` ).

Вы можете очистить сохраненные cookie-файлы для данного домена на [панели инструментов](#network-summary) (кнопка**Очистить "cookie** "). 

### Времени показа

Вкладка " **время** " содержит временную шкалу сетевых событий, связанных с загрузкой выбранного ресурса. Это аналогично сведениям, находящихся в столбце *временная шкала* в [списке сетевых запросов](#network-request-list), но также включает события, ведущие к запросу, отправляемому по сети, например время ожидания (*остановлено*) в очереди запросов, разрешение DNS и установление соединения по протоколу TCP. 

![Вкладка "время" в области "сведения о запросе"](./media/network_details_timings.png)

Помечаются перенаправления между другими ресурсами и нажатием ссылки в области [сведений о](#request details) сетевом запросе будет установлено фокус на этот ресурс.

Время задержки сети не влияет на ресурсы, загружаемые из кэша, поэтому не отображается диаграмма *синхронизации* сети.

![Перенаправленный ресурс, загруженный из кэша](./media/network_details_timings_cache_redirect.png)

Ниже указаны различные сетевые события, которые могут отображаться для определенного ресурса в хронологическом порядке.

#### Блокированы

Время, потраченное на ожидание доступа к сетевому подключению в очереди запросов. Для HTTP 1.0/1.1 Microsoft EDGE поддерживает не более шести (6) одновременных подключений TCP на каждое имя узла. 

#### Разрешение (DNS)

Время, потраченное на поиск IP-адреса узла ресурса в DNS ([системе доменных имен](https://en.wikipedia.org/wiki/Domain_Name_System)).

#### Подключение (TCP)

Время, затраченное на установку соединения TCP ([протокол управления передачей](https://en.wikipedia.org/wiki/Transmission_Control_Protocol)).

#### ТЕХНОЛОГИЮ

Время, затраченное на согласование соединения SSL ([Secure Socket Layer](https://en.wikipedia.org/wiki/Transport_Layer_Security)) с [прокси-сервером](https://en.wikipedia.org/wiki/Proxy_server) для узла.

#### Отправка

Время, затраченное на отправку запроса ресурса.

#### Ожидание (TTFB)

Время, потраченное на ожидание первого байта ответа от хост-сервера ("время до первого байта" или *TTFB*).

#### Скачать

Время, потраченное на чтение ответа с сервера.

## Известные проблемы

### Не удалось запустить агент сетевого семейства.

Если отображается это сообщение об ошибке: **агенту сетевого семейства сети не удалось запустить** средство "сеть", выполните указанные ниже действия, чтобы устранить проблему.

1. Нажмите клавишу `Windows Key`  +  `R` .

2. В диалоговом окне Выполнить введите **Services. msc**.
![Известные проблемы – 1](./media/known_issues_1.PNG)

3. Найдите **стандартную службу сборщика центра диагностики Microsoft (R)** и щелкните ее правой кнопкой мыши.
![Известные проблемы – 2](./media/known_issues_2.PNG)

4. Перезапустите **стандартную службу сборщика центра диагностики Microsoft (R)**.
![Известные проблемы – 3](./media/known_issues_3.PNG)

5. Закройте инструменты разработчика Microsoft EDGE и вкладку. Откройте новую вкладку, перейдите на нужную страницу и нажмите клавишу `F12` .

6. Теперь вы должны увидеть индикатор воспроизведения рядом с командой **сеть** и сетевые запросы для веб-страницы.
![Известные проблемы — 4](./media/known_issues_4-network.PNG)

Вы по-прежнему столкнулись с проблемами? Пожалуйста, отправьте нам свой отзыв с помощью значка " **Отправить отзыв** "!
