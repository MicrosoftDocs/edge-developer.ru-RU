# Предотвращение отслеживания в Microsoft Edge (Chromium)  

Функция предотвращения отслеживания в Microsoft Edge защищает пользователей от отслеживания в Интернете, ограничивая возможность отслеживания доступом к хранилищу в браузере и сети.  Она встроена в обещание конфиденциальности браузера Microsoft [Edge,][MicrosoftEdgeBrowserPrivacyPromise] а также обеспечивает отсутствие влияния по умолчанию на совместимость веб-сайтов или на экономию веб-сайтов.  

В настоящее время Microsoft Edge предлагает пользователям три уровня предотвращения отслеживания, которые выбираются при переходе к `edge://settings/privacy` .  

![Три параметра предотвращения отслеживания][ImageThreeSettingsTrackingPrevention]  

1.  **Базовый** — наименее строгий уровень предотвращения отслеживания, разработанный для пользователей, которые пользуются персонализированными объявлениями и не против отслеживания в Интернете.  Базовый защищает пользователей только от вредоносных отслеживаний, таких как отпечатки пальцев и криптоминеры.  
1.  **Сбалансированный (по умолчанию)** — уровень по умолчанию для предотвращения отслеживания, предназначенный для пользователей, которые хотят видеть менее юные объявления, которые следуют за ними в Интернете во время просмотра.  Сбалансированная цель состоит в блокировании средств отслеживания с сайтов, с которые пользователи никогда не участвуют, а также минимизации риска проблем совместимости в Интернете.  
1.  **Строгая** — самый строгий уровень предотвращения отслеживания, разработанный для пользователей, которые не против совместимости веб-сайтов для обеспечения максимальной конфиденциальности.  

Функция предотвращения отслеживания в Microsoft Edge состоит из трех основных компонентов, которые вместе определяют, следует ли классифицировать определенный ресурс с веб-сайта как отслеживающие и блокировать.  Компоненты:  

1.  **Классификация** — способ, с которой Microsoft Edge определяет, принадлежит ли URL-адрес для отслеживания.  
1.  **Enforcement** — действия, принятые для защиты пользователей Microsoft Edge от URL-адресов, классифицированных как отслеживатели.  
1.  **Меры защиты — механизмы,** предоставляемые для обеспечения того, чтобы указанные пользователем избранные сайты по-прежнему работали, обеспечивая надежную защиту по умолчанию.  

Каждый из компонентов подробно изучается и подробно объясняется на этой странице.  

## Категория  

Первым компонентом функции предотвращения отслеживания в Microsoft Edge является классификация.  Чтобы классифицировать сетевые отслеживания и группировать их по категориям, Microsoft Edge использует списки [защиты][|::ref1::|Main] от отключения отслеживания [с открытым кодом.][GitHubDisconnectMeTrackingProtection]  Списки доставляются через компонент "Списки защиты доверия", который можно просмотреть `edge://components` по  После загрузки списки хранятся на диске, где их можно использовать для определения классификации определенного URL-адреса.  

Чтобы определить, считается ли URL-адрес отслеживанием системой классификации в Microsoft Edge, проверяется ряд имен хостов, начиная с точного совпадения, а затем проверяя наличие частичных совпадений для четырех меток за пределами домена верхнего уровня.  

> **Пример**:  
> 
> URL-адрес: `https://a.subdomain.of.a.known.tracker.test/some/path`  
> 
> Протестные имена хостов:  
> 
> *   `a.subdomain.of.a.known.tracker.test`  
> *   `of.a.known.tracker.test`  
> *   `a.known.tracker.test`  
> *   `known.tracker.test`  
> *   `tracker.test`  

Если любое из этих имен хостов [][|::ref2::|Main] совпадает с именем хоста в списках отключения, [][GitHubDisconnectMeTrackingProtection]Microsoft Edge продолжает оценку действий по предотвращению отслеживания пользователей.  

## Принудительное применение  

Чтобы обеспечить защиту от отслеживания действий в Интернете, Microsoft Edge принимает два действия по защите от секретных отслеживаний:

*   **Ограничение доступа к хранилищу** . Если известный ресурс отслеживания пытается получить доступ к любому веб-хранилищу, где он может пытаться сохранить данные о пользователе, Microsoft Edge блокирует этот доступ.  Это включает ограничение возможности для этого устройства отслеживания получать или устанавливать файлы cookie, а также получать доступ к API хранилища, таким как `IndexedDB` и `localStorage` .  
*   **Блокировка** загрузки ресурсов . Если на веб-сайт загружается известный ресурс отслеживания, Microsoft Edge может заблокировать эту нагрузку, прежде чем запрос достигнет сети в зависимости от влияния загрузки на совместимость и параметра предотвращения отслеживания, задаемого пользователем.  Заблокированные загрузки могут включать сценарии отслеживания, пиксели, iframes и другие.  Это предотвращает потенциальное отправление данных в домен отслеживания и может даже повысить время загрузки и производительность страницы в качестве побочных эффектов.  

Пользователь может щелкнуть значок "Сведения о странице" в левой части адресной панели, чтобы узнать, какие отслеживатели были заблокированы на определенной странице: 

![Заблокированные отслеживатели во flyout информации о странице][ImageBlockedTrackersPageInfoFlyout]  

Применение этих мер зависит от уровня предотвращения отслеживания, выбранного пользователем, и от мер, которые могут применяться.  

## Меры по смягчению последствий  

Чтобы обеспечить максимальное сохранение совместимости веб-сайтов, Microsoft Edge имеет три меры, которые помогают сбалансировать принудительных режимов в определенных ситуациях.  Это меры [для связи](#org-relationship-mitigation)организации, меры для org [Engagement](#org-engagement-mitigation)и [список CompatExceptions.](#the-compatexceptions-list)  

Прежде чем перегруженными в меры, стоит определить концепцию "Организация" или "Организация" для краткости.  [Disconnect][|::ref3::|Main] также поддерживает список [][GitHubDisconnectMeTrackingProtectionEntitiesJson]entities.js, в который определяются группы URL-адресов, которые принадлежат той же родительской организации или компании.  Функция предотвращения отслеживания в Microsoft Edge [](#org-relationship-mitigation) использует этот список как в средстве снижения связи организации, так и в средстве "Org [Engagement",](#org-engagement-mitigation) чтобы свести к минимуму возникновение проблем совместимости, вызванных отслеживанием предотвращения, затрагивающих запросы между организации.  

### Увязание связи организации  

Некоторые популярные веб-сайты обслуживают веб-сайты и сети доставки содержимого \(CDNs\) для обслуживания статических ресурсов и контента на этих сайтах.  Чтобы предотвратить отслеживание таких сценариев, Microsoft Edge освобождает сайт от предотвращения отслеживания, когда сайт делает запросы сторонних лиц на другие сайты, которые принадлежат той же родительской организации \(как определено в disconnect [entities.json list][GitHubDisconnectMeTrackingProtectionEntitiesJson]\).  Это лучше всего проиллюстрировано в примере.  

> **Пример.**
> 
> Организация с именем Org1 владеет доменами и, как определено в списке `org1.test` `org1-cdn.test` disconnect entities.jsв [списке.][GitHubDisconnectMeTrackingProtectionEntitiesJson]  Представьте, что классифицируется как устройство отслеживания и к ней обычно применяются меры предотвращения `org1-cdn.test` отслеживания.  Если пользователь посещает сайт и пытается загрузить ресурс с него, Microsoft Edge не принимает никаких действий по принулителям к запросам, даже если это не `https://org1.test` `https://org1-cdn.test` `org1-cdn.test` URL-адрес от первого пользователя.  Однако если другой URL-адрес, который не является частью организации Org1, попытается загрузить этот же ресурс, запрос будет подвергаться принулизации, так как он не является частью одной и той же организации.  
> 
> Несмотря на то, что это снижает уровень защиты от слежения за сайтами, принадлежащими одной организации, маловероятно, что это представляет высокий риск конфиденциальности, так как такие организации могут определить, к как сайтам и ресурсам, к которым вы получили доступ, а также использовать внутренние `https://org1.test` `https://org1-cdn.test` внутренние данные.  

### Меры по унижению вовлечения в организацию  

Решение о задействовании организации было создано, чтобы свести к минимуму риски совместимости, введенные путем отслеживания предотвращения, обеспечивая тем самым, что сайты, которые принадлежат организациям, которые достаточно вовлечены пользователями, продолжают работать в Интернете.  Он использует [][ChromiumDesignDocsSiteEngagement] вовлечение сайта, чтобы принудительных усилий всякий раз, когда пользователь установил текущие отношения \(в настоящее время определяется оценкой вовлеченности сайта 4,1 или более\) с заданным сайтом.  Этот пример также лучше всего проиллюстрировать в примере:

> **Пример.**
> 
> Организация с именем Social Org владеет доменами `social.example` и `social-videos.example` .
> 
> Считается, что пользователи имеют отношение с социальной организацией, если они установили показатель вовлеченности сайта в 4,1 или более с любым из доменов, владельцем которые являются социальные организации.
> 
> Если другой сайт, включаемый в себя сторонний контент \(например, внедренное видео из \) из любого из доменов, владельцем котором является social Org, который обычно ограничивается отслеживанием мер предотвращения, на сайте не будет отслеживания мер предотвращения, если показатель вовлеченности сайта пользователя с доменами, которые принадлежат социальной организации, не превышает пороговое `https://content-embedder.example` `social-videos.example` значение.
> 
> Если сайт не принадлежит организации, пользователь должен установить показатель вовлеченности сайта в 4,1 или более непосредственно, прежде чем любые блоки доступа к хранилищу или ресурсов, наложенные предотвращением отслеживания, будут ослаблены.

Меры по защите от вовлечения в организацию в настоящее время применяются только в сбалансированном режиме, поэтому Microsoft Edge предлагает наиболее высокий уровень защиты для пользователей, которые решили использовать Strict.

### Список CompatExceptions  

На основе последних отзывов пользователей, полученных корпорацией Майкрософт, Microsoft Edge ведет небольшой список сайтов (большинство из которых находятся в категории "Отключение содержимого"),которые были прерваны из-за предотвращения отслеживания несмотря на наличие двух вышеперечисленных мер. Сайты в этом списке не могут отслеживать меры предотвращения.  Список можно найти на диске в [расположениях,](#determining-whetherhow-a-particular-url-is-classified) описанных ниже.  Пользователи могут переопределять записи на нем с помощью **параметра "Блокировать"** в `edge://settings/content/cookies` .

Чтобы не сохранять этот список в будущем, корпорация Майкрософт в настоящее время работает над [API][GitHubMsExplainersStorageAccessApi] доступа к хранилищу в базе кода Chromium.  [API][GitHubMsExplainersStorageAccessApi] доступа к хранилищу позволяет разработчикам сайтов запрашивать доступ к хранилищу у пользователей напрямую, обеспечивая пользователям больше прозрачности в отношении того, как параметры конфиденциальности влияют на их работу при просмотре, и предоставляя разработчикам элементов управления сайта для быстрого и интуитивного разблокировки.

После реализации [API][GitHubMsExplainersStorageAccessApi] доступа к хранилищу корпорация Майкрософт амортирует список CompatExceptions и обратилась к затронутым сайтам, чтобы они знали о проблемах и запросили использование [API][GitHubMsExplainersStorageAccessApi] доступа к хранилищу в будущем.  

## Текущее поведение предотвращения отслеживания  

В следующей таблице показаны принудительных действий и мер, которые применяются к каждой категории классифицированного отслеживания в Microsoft Edge.  

*   Вверху находятся категории отслеживания, определенные категориями списка защиты от [отключения.][GitHubDisconnectTrackingProtectionCategories]  
*   Вдоль левой стороны находятся три уровня предотвращения отслеживания в Microsoft Edge \(Basic, Balanced и Strict\).  
*   Буква `S` указывает, что доступ к хранилищу заблокирован.  
*   Буква указывает, что доступ к хранилищу и загрузка `B` ресурсов \(например, сетевые запросы\) заблокированы.  
*   Дефис \( \) указывает, что блок не применяется ни к доступу к хранилищу, ни `-` к загрузке ресурсов.  

| | Ветвь | Аналитика | Содержимое | Cryptomining | Отпечатки пальцев | Социальные сети | Other | Одно и то же увязание организации | Меры по унижению вовлечения в организацию |  
| - | - | - | - | - | - | - | - | - | - | - |  
| **Базовый** | - | - | - | B | B | - | - | Включено | Н/Д |  
| **Сбалансированная** | S | - | S | B | B | S | S | Включена | Включена |  
| **"Строгий"** | B | B | S | B | B | B | B | Enabled | Отключено |  

> [!NOTE]
> Меры по смягчению последствий для вовлечения в организацию не применяются к категориям Cryptomining или Fingerprinting.  

> [!TIP]
> Строгий режим блокирует больше нагрузки ресурсов, чем сбалансированный.  Блокировка большего количества загрузок ресурсов может привести к блокированию в строгом режиме меньшего количества запросов отслеживания, чем "Сбалансировано", так как отслеживающие, делающие запросы, никогда не загружаются.  

> [!NOTE]
> Столбец "Отпечатки пальцев" в [текущем](#current-tracking-prevention-behavior) поведении предотвращения отслеживания относится к отслеживанию, которые находятся в списке отпечатков пальцев в дополнение к другому списку.  Отслеживания, которые отображаются только в списке отпечатков пальцев, считаются не вредоносными отпечатками пальцев и не блокируются.

### Поведение InPrivate  

В Microsoft Edge 79 поведение по умолчанию применялось в Режиме Strict в InPrivate.  В Microsoft Edge 80 это поведение было заменено переключателем, который позволяет пользователям решать, применять ли строгий режим защиты или сохранять свои обычные параметры при просмотре `edge://settings/privacy` InPrivate.  

## Определение классификации определенного URL-адреса  

Самый простой способ определить, классифицируется ли определенный URL-адрес как известный отслеживатель, — выполнить следующие действия.  

1.  Откройте DevTools и перейдите на вкладку "Консоль".  
1.  Перезагрузите страницу.  
    1.  Сначала может потребоваться очистить файлы cookie и **другие** данные сайта, чтобы сбросить показатели вовлеченности сайта и обеспечить полностью чистую доску.  
1.  Найми все прочитаные `Tracking Prevention blocked access to storage for <URL>` сообщения.  
    1.  Вы можете развернуть сообщения, чтобы увидеть отдельные заблокированные URL-адреса.  
1.  Если вам нужно определить категорию определенного заблокированного сайта, самый простой способ сделать это — найти ее в списке отключения [services.js.][GitHubDisconnectTrackingProtectionCategories]  Записи в алфавитном порядке, поэтому прокрутка до верхней части блока записей сайта позволяет найти определенную категорию для определенного сайта.  

> [!TIP]
> Если вам нужен доступ к спискам предотвращения отслеживания, которые хранятся на диске, их можно найти в одном из двух мест.  
>
> **Обновления на основе компонентов** — списки, скачиваемые из компонента "Списки защиты доверия"  
>
> Windows: `%LOCALAPPDATA%\Microsoft\Edge <OptionalChannelName>\User Data\Trust Protection Lists`  
>
> macOS: `~/Library/Application Support/Microsoft Edge <OptionalChannelName>/Trust Protection Lists`  
>
> **Каталог установки —** списки, в комплекте с установщиком Microsoft Edge.  Если вы выбрали другой каталог установки, точные пути могут быть другими.  
>
> Windows: `%PROGRAMFILES(x86)%\Microsoft\ Edge <OptionalChannelName>\Application<Version>\Trust Protection Lists`  
>
> macOS: `/Applications/Microsoft Edge.app/Contents/Frameworks/Microsoft Edge Framework.framework/Libraries/Trust Protection Lists`  

## Вопросы и ответы  

В следующем разделе представлены ответы на часто задающие вопросы о функции предотвращения отслеживания в Microsoft Edge.  

**Есть ли способ заблокировать или разрешить определенные отслеживания в целях отладки?**  

В настоящее время Microsoft Edge предоставляет возможность отключить запуск средств предотвращения отслеживания только на указанном сайте.  Доступ к этому параметру можно получить с помощью flyout информации о странице или через `edge://settings/privacy/trackingPreventionExceptions` страницу.  

При этом параметры **** блокировки и запрета на странице могут использоваться для того, чтобы разрешить или запретить определенным доменам доступ к хранилищу, например файлам cookie и другим механизмам хранения в **** `edge://settings/content/cookies` браузере.  Это полезно для отладки проблем сайта, вызванных отслеживанием мер предотвращения блокирования доступа к хранилищу для определенного сайта.
