# <a name="javascript-representation-of-windows-runtime-types"></a>Представление типов среды выполнения Windows в JavaScript  

[!INCLUDE [deprecation-note](../includes/legacy-edge-note.md)]  

В следующей таблице описывается, как JavaScript представляет типы времени запуска Windows.  

> [!IMPORTANT]
> Функции windows Runtime недоступны для приложений, которые работают в Internet Explorer.  

## <a name="windows-runtime-types-in-javascript"></a>Типы времени запуска Windows в JavaScript  

| Тип времени запуска Windows | Представление JavaScript | Описание |  
|:--- |:--- |:--- |  
| `UInt8` | `Number` | Windows Runtime `Uint8` — это неподписаный 8-битный целый ряд, представленный в качестве номера JavaScript.  Значение JavaScript сначала преобразуется в номер JavaScript, а затем `ToUint32` следует процесс.  Если значение номера JavaScript находится за пределами диапазона Uint8, в результате будет применен модуль 2^8.  |  
| `Int32` | `Number` | Windows Runtime — это `Int32` подписанный 32-битный целый ряд, представленный как номер JavaScript.  Значение JavaScript сначала преобразуется в номер JavaScript, а затем `ToInt32` следует процесс.  Если значение номера JavaScript находится за пределами диапазона, в результате будет применен модуль `Int32` 2^16.  |  
| `Int64` | `Number` | Windows Runtime — это подписанный 64-битный набор, представленный в качестве стандартного номера, если он попадает в диапазон `Int64` \[-2^53, 2^53\].  Если он выпадает за пределы этого диапазона, он представляется в качестве значения числа, которое имеет настраиваемый запасной магазин, который поддерживает полные 64 бита данных в наборе.  Все математические операции по этим пользовательским значениям номеров Int64 приводят к преобразованию значения в стандартное число в диапазоне \[-2^53, 2^53\].  Если значение находится за пределами этого диапазона, повышается ошибка типа.  Исключениями для этого процесса преобразования являются операции , и , которые сравнивают аргументы на основе полных 64 битов, когда прошло два представлены `==` `===` `SameValue` `RelationalComparison` 64-битные значения.  Если любой аргумент является стандартным номером JavaScript, эти операции также преобразуют аргумент в номер JavaScript перед сравнением.  Значение JavaScript напрямую назначено, если оно само по себе является значением Int64; в противном случае результат применения преобразования значения передается в `ToInteger` Windows Runtime.  Если принуждение типа не удается, возвращается исключение.  |  
| `Uint64` | `Number` | Windows Runtime — это неподписаный 64-битный набор, представленный в качестве стандартного номера, если он попадает в диапазон `Uint64` \[0, 2^53\].  Если он выпадает за пределы этого диапазона, он представляется как номер, который имеет настраиваемый запасной магазин, который поддерживает полные 64 биты неподписаных данных.  Все математические операции по этим настраиваемому значениям Uint64 приводят к преобразованию значения в стандартное число в диапазоне \[0, 2^53\].  Если значение находится за пределами этого диапазона, повышается ошибка типа.  Исключениями из этого процесса преобразования являются операции , и операции, которые сравнивают аргументы на основе полных 64 битов при сдав два `==` `===` `SameValue` `RelationalComparison` 64-битных значения.  Если любой аргумент является стандартным номером JavaScript, эти операции также преобразуют аргумент в номер JavaScript перед сравнением.  Значение JavaScript напрямую назначено, если оно само по себе является значением Uint64; в противном случае результат применения преобразования к значению с последующим принятием модуля `ToInteger` 2^52 передается в Windows Runtime.  Если преобразование типа не удается, возвращается исключение.  |  
| `Single` | `Number` | Windows Runtime — это 32-разрядный номер с плавающей точкой, представленный в качестве номера JavaScript, выбирая максимально близкое значение с двойной точностью к `Single` одноточному значению.  Значение JavaScript преобразуется в номер JavaScript, а затем проверяется диапазон, чтобы убедиться, что значение может быть представлено в одноточном 32-разрядной номере плавающей точки IEEE 754.  Если проверка преобразования или диапазона не удается, возвращается ошибка маршалинга.  |  
| `Double` | `Number` | Время запуска Windows `Double` — это 64-битный номер с плавающей точкой.  `ToNumber` используется для преобразования времени запуска Windows `Double` в номер JavaScript.  Если преобразование не удается, возвращается ошибка маршалинга.  |  
| `Boolean` | `Boolean` | Время запуска Windows — это 8-битное значение, где ноль, а любое значение, кроме нуля, представлено `Boolean` `false` как `true` `Boolean` JavaScript.  Значение JavaScript сначала преобразуется в JavaScript `Boolean` путем `ToBoolean` .  Это означает, что функция windows Runtime, объявленная так, `void func(BOOL);` как может быть написана в JavaScript как `obj.func("test");` .  Функция windows Runtime называется с `BOOL` параметром, переданным как `TRUE` .  Если преобразование не удается, возвращается ошибка маршалинга.  |  
| `Char16` | `String` | Windows Runtime — это 16-разрядный символ Юникод, представленный в качестве строки JavaScript, которая содержит `Char16` один символ.  Значение JavaScript преобразуется в строку JavaScript, а затем проверяется, чтобы убедиться, что строка имеет длину `ToString` только одного символа.  Затем один символ передается в качестве `Char16` значения.  Если проверка преобразования или длины не удается, возвращается ошибка маршалинга.  |  
| `String` | `String` | Время запуска Windows `String` — это неоменяемая последовательность `Char16` объектов.  Строки представлены как объекты `String` JavaScript.  Строки времени запуска Windows не имеют разных значений для `null` и пустой строки \("\").  `null` и пустые значения строки преобразуются в пустую строку JavaScript.  **ПРИМЕЧАНИЕ.** Когда JavaScript передается параметру Windows Runtime, который ожидает строку, он создает строковую строку "null", а не пустую строку `null` `null` \(""\").  Строки, переданные в Windows Runtime, всегда должны быть моментально переданы пустой строке.  |  
| `Enumeration` | `Object` | Время запуска Windows — это 32-битный набор \(подписанный или неподписаный\), который имеет связанный набор `Enumeration` именных констант.  В JavaScript переумеление представлено как объект, содержащий поле только для чтения для каждого названного значения.  Значения переопределения преобразуются в номера JavaScript, как было определено ранее для `Int32` и `UInt32` .  При преобразовании значения JavaScript в переописание времени запуска Windows оно преобразуется, усечено и проверяется диапазон, как описано ранее.  Однако итоговая величина не проверяется с заданными значениями, указанными в переумериях.  |  
| `Structure` | `Object` | Время запуска Windows `Structure` — это разнородное собрание именованных полей данных.  В JavaScript структура представлена как объект JavaScript, содержащий свойство с именем для каждого поля в структуре.  Если преобразование значения структуры не удается, возвращается ошибка маршалинга.  Поля в объекте JavaScript, который не имеет эквивалентов в структуре windows Runtime, игнорируются.  **ПРИМЕЧАНИЕ.** Структура времени запуска Windows не может быть мгновенной в JavaScript с помощью ключевого `new` слова.  |  
| `Array` | `Array` | Время запуска Windows `Array` преобразуется в массив JavaScript.  Однако массивы windows Runtime не являются resizable, поэтому некоторые операции массива JavaScript не возможны.  Внутреннее свойство преобразованного массива не заданной для массива, так как это не разрешено спецификацией `[[Class]]` ECMAScript 5.  Это означает, `Array.isArray(v)` что возвращает `false` .  Значение JavaScript преобразуется в массив времени запуска Windows следующим образом: Если это значение или , оно преобразуется `null` `undefined` в родной `null` .  Если его внутреннее свойство " Array", оно копируется в собственный массив, и ссылка на этот массив `[[Class]]` передается.  Если это преобразованный массив, как описано выше, то будет передана родной массив.  **ПРИМЕЧАНИЕ.** Передача значения массива JavaScript методу windows Runtime вызывает полную копию массива.  |  
| Делегирование \(функция вызова\) | `Function` | Делегирование времени запуска Windows — это ссылка на один метод.  Делегат времени запуска Windows представлен в JavaScript в качестве JavaScript, который гарантированно будет вызван `Function` в нужном потоке.  При вызове аргументы преобразуются в эквивалентные типы `Function` параметров windows runtime.  Если аргументов JavaScript меньше, чем параметров делегирования, вызов `in` делегата не удается.  Если в делегате больше аргументов JavaScript, чем параметров, дополнительные аргументы `in` JavaScript игнорируются.  После вызова делегата параметры преобразуются в `out` типы JavaScript и возвращаются.  Если родной объект функции JavaScript преобразуется в делегата времени запуска Windows, он завернут в делегата времени запуска Windows соответствующего типа.  При вызове делегата `in` параметры преобразуются в типы JavaScript.  После вызова делегата возвращаемая величина преобразуется в `out` параметры делегата.  |  
| Интерфейс | `Object` | Интерфейсы и группы интерфейсов не представлены непосредственно в JavaScript в качестве объектов.  Однако интерфейсы представлены в качестве типов параметра и возврата методов windows Runtime.  Экземпляр интерфейса windows Runtime преобразуется следующим образом:<br /> <br /> 1. Если имеется допустимый класс времени работы, представляете объект в качестве экземпляра этого класса.<br /> 2. Если не существует допустимого класса выполнения, представляете объект в качестве экземпляра неназванного класса времени выполнения, который реализует именно интерфейс, который экземпляр, как известно, реализует, плюс его необходимые интерфейсы.<br /> <br /> Значение JavaScript протестировано, чтобы определить, является ли это экземпляром класса времени работы или интерфейса.  Если это так, а исходное значение времени выполнения Windows реализует интерфейс, объект передается в Windows Runtime.  |  
| Классы runtime | `Object` | Объекты в windows Runtime могут быть экземплярами классов выполнения, реализуя один или несколько интерфейсов.  Объекты windows Runtime представлены в JavaScript как объекты.  Объединение методов, свойств и событий, определенных во всех реализованных интерфейсах класса, представляет названные свойства в прототипе объекта JavaScript.  Пользователи JavaScript могут напрямую получить доступ к любому члену класса.  Объекты windows Runtime имеют прототип, содержащий всех участников из всех реализованных интерфейсов класса выполнения.  |
