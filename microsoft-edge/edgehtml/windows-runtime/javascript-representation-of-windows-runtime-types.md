# Представление типов среды выполнения Windows в JavaScript  

[!INCLUDE [deprecation-note](../includes/legacy-edge-note.md)]  

В следующей таблице описано, как JavaScript представляет типы времени работы Windows.  

> [!IMPORTANT]
> Функции времени работы Windows недоступны для приложений, которые работают в Internet Explorer.  

## Типы времени работы Windows в JavaScript  

| Тип времени работы Windows | Представление JavaScript | Описание |  
|:--- |:--- |:--- |  
| `UInt8` | `Number` | Windows Runtime `Uint8` — это неподписаное 8-битное число, представляемые в качестве номера JavaScript.  Значение JavaScript сначала преобразуется в число JavaScript, а затем процесс `ToUint32` следует за ним.  Если значение номера JavaScript находится за пределами диапазона Uint8, к результату будет применено модуло 2^8.  |  
| `Int32` | `Number` | Время работы Windows `Int32` — это 32-битное 32-битное число, представляемые в качестве номера JavaScript.  Значение JavaScript сначала преобразуется в число JavaScript, а затем процесс `ToInt32` следует за ним.  Если значение номера JavaScript находится за пределами диапазона, к результату будет применено модуло `Int32` 2^16.  |  
| `Int64` | `Number` | Windows Runtime — это подписанное 64-битное число, представляемые как стандартное число, если оно попадает в диапазон `Int64` \[-2^53, 2^53\].  Если оно попадает за пределы этого диапазона, оно представляется как числовая величина с пользовательским хранилищем, которое поддерживает полные 64 бита полных данных.  Все математические операции с этими пользовательскими числами Int64 приводят к преобразованию значения в стандартное число в диапазоне \[-2^53, 2^53\].  Если значение находится за пределами этого диапазона, вызывается ошибка типа.  Исключениями из этого процесса преобразования являются операции , , и, которые сравнивают аргументы на основе полных 64 бит, когда передаются два представленных `==` `===` `SameValue` `RelationalComparison` 64-битных значения.  Если один из аргументов является стандартным номером JavaScript, эти операции также преобразуют его в число JavaScript перед сравнением.  Значение JavaScript напрямую назначено, если оно само является значением Int64; в противном случае результат применения преобразования к значению передается `ToInteger` в итоге в итоге.  Если не удается привести тип, возвращается исключение.  |  
| `Uint64` | `Number` | Windows Runtime — это неподписаное 64-битное число, представляемые как стандартное число, если оно попадает в диапазон `Uint64` \[0, 2^53\].  Если он попадает за пределы этого диапазона, он представлен как число с пользовательским хранилищем, которое поддерживает полные 64 бита неподписаных полных данных.  Все математические операции с этими пользовательскими значениями Uint64 приводят к преобразованию значения в стандартное число в диапазоне \[0, 2^53\].  Если значение находится за пределами этого диапазона, вызывается ошибка типа.  Исключением из этого процесса преобразования являются операции , , и, которые сравнивают аргументы на основе полных 64 битов, когда передаются два `==` `===` `SameValue` `RelationalComparison` 64-битных значения.  Если один из аргументов является стандартным номером JavaScript, эти операции также преобразуют его в число JavaScript перед сравнением.  Значение JavaScript напрямую назначено, если оно само является значением Uint64; в противном случае результат применения преобразования к значению с последующим принятием модуля 2^52 передается в порядок работы `ToInteger` Windows.  Если преобразование типа не удается, возвращается исключение.  |  
| `Single` | `Number` | Windows Runtime — это 32-разрядное число с плавающей за точкой, представленное в качестве числа JavaScript путем выбора ближайшего значения двойной точности к значению с одной `Single` точностью.  Значение JavaScript преобразуется в число JavaScript, а затем проверяется диапазон, чтобы убедиться, что значение может быть представлено в 32-разрядной 32-разрядной точке IEEE 754 с плавающей за точкой.  Если не удается проверить преобразование или диапазон, возвращается ошибка маршалирования.  |  
| `Double` | `Number` | Windows Runtime `Double` — это 64-битное число с плавающей за точкой.  `ToNumber` используется для преобразования времени работы Windows `Double` в число JavaScript.  Если преобразование не удается, возвращается ошибка маршалинга.  |  
| `Boolean` | `Boolean` | 8-битное значение— это 8-битное значение, где ноль и любое другое значение, кроме нуля, представлено `Boolean` `false` как `true` `Boolean` JavaScript.  Значение JavaScript сначала преобразуется в JavaScript `Boolean` с помощью `ToBoolean` .  Это означает, что функция времени работы Windows, объявленная как объявленная, может `void func(BOOL);` быть написана на JavaScript как `obj.func("test");` .  Функция времени работы Windows вызвана с `BOOL` переданным параметром `TRUE` .  Если преобразование не удается, возвращается ошибка маршалинга.  |  
| `Char16` | `String` | Windows Runtime — это 16-разрядный символ Юникода, представленный в качестве строки `Char16` JavaScript, которая содержит один символ.  Значение JavaScript преобразуется в строку JavaScript, а затем проверяется, чтобы строка была длиной всего `ToString` один символ.  Затем в качестве значения передается один `Char16` символ.  Если не удается проверить преобразование или длину, возвращается ошибка маршалирования.  |  
| `String` | `String` | Windows Runtime `String` — это не изменяемая последовательность `Char16` объектов.  Строки представлены в качестве объектов `String` JavaScript.  Строки во время работы Windows не имеют разных значений для `null` пустой строки \(""\).  `null` и пустые строки преобразуются в пустую строку JavaScript.  **ПРИМЕЧАНИЕ.** Когда JavaScript передается параметру времени runtime Windows, который ожидает строку, он создает строку "null", а не пустую строку `null` `null` \(""\).  Строки, переданные в пустую строку, всегда должны быть переданы в пустую строку.  |  
| `Enumeration` | `Object` | `Enumeration`32-битное integer \(signed или unsigned\) — это 32-битное integer \(signed или unsigned\), со связанным набором именуемой констант.  В JavaScript enumeration is represented as an object that contains a read-only field for each named value.  Значения enumeration преобразуются в числа JavaScript, как было определено ранее для `Int32` и `UInt32` .  Когда значение JavaScript преобразуется в индекс времени работы Windows, оно преобразуется, усечено и диапазон проверяется, как описано выше.  Однако итоговые значения не проверяются с заданными именоваными значениями, указанными в этом значении.  |  
| `Structure` | `Object` | Windows Runtime `Structure` — это разнородная коллекция именованных полей данных.  В JavaScript структура представлена как объект JavaScript, который содержит именуемого свойства для каждого поля в структуре.  Если преобразование какого-либо значения структуры не удается, возвращается ошибка маршалирования.  Поля в объекте JavaScript, которые не имеют эквивалентов в структуре времени работы Windows, игнорируются.  **ПРИМЕЧАНИЕ.** Структуру времени работы Windows нельзя использовать в JavaScript с ключевым `new` словом.  |  
| `Array` | `Array` | Время работы Windows `Array` преобразуется в массив JavaScript.  Однако массивы в среде разработки Windows не могут меняться, поэтому некоторые операции с массивами JavaScript недоступны.  Внутреннее свойство преобразованного массива не имеет свойства "Array", так как это запрещено спецификацией `[[Class]]` ECMAScript 5.  Это означает, `Array.isArray(v)` что возвращает `false` .  Значение JavaScript преобразуется в массив времени runtime Windows следующим образом: если это значение или , оно преобразуется в `null` `undefined` родной `null` .  Если его `[[Class]]` внутреннее свойство — "Array", оно копируется в собственный массив, и передается ссылка на этот массив.  Если это преобразованный массив, как описано выше, передается его в основном массиве native.  **ПРИМЕЧАНИЕ.** Передача значения массива JavaScript методу времени работы Windows приводит к полному копированию массива.  |  
| Delegate \(function callback\) | `Function` | Делегат в времени работы Windows является ссылкой на один метод.  Делегат в среде запуска Windows представлен в JavaScript как JavaScript, который гарантированно будет вызван в `Function` соответствующем потоке.  При вызове аргументы преобразуются в эквивалентные типы параметров `Function` времени работы Windows.  Если аргументов JavaScript меньше, чем параметров делегата, вызов `in` делегата не удастся.  Если в делегате больше аргументов JavaScript, чем параметров, дополнительные аргументы `in` JavaScript игнорируются.  После вызова делегата параметры преобразуются в типы `out` JavaScript и возвращаются.  Если машинный объект функции JavaScript преобразуется в делегата времени работы Windows, он преобразуется в делегат времени запуска Windows соответствующего типа.  При вызове делегата `in` параметры преобразуются в типы JavaScript.  После вызова делегата возвращаемая величина преобразуется в `out` параметры делегата.  |  
| Интерфейс | `Object` | Интерфейсы и группы интерфейсов не представлены непосредственно в JavaScript как объекты.  Однако интерфейсы представлены в качестве параметров и возвращаемого типа методов времени работы Windows.  Экземпляр интерфейса времени работы Windows преобразуется следующим образом:<br /> <br /> 1. Если существует допустимый класс времени работы, представить объект в качестве экземпляра этого класса.<br /> 2. Если допустимого класса времени выполнения нет, представить объект как экземпляр неименоваемого класса времени выполнения, который реализует в точности интерфейс, реализуемого экземпляром, а также его необходимые интерфейсы.<br /> <br /> Значение JavaScript тестировано, чтобы определить, является ли это экземпляром класса времени работы или интерфейсом.  Если это так, а исходное значение времени выполнения Windows реализует интерфейс, объект передается в начальную начальную окна windows.  |  
| Классы времени работы | `Object` | Объектами в времени выполнения Windows могут быть экземпляры классов времени выполнения, которые реализуют один или несколько интерфейсов.  Объекты времени запуска Windows представлены в JavaScript как объекты.  Объединение методов, свойств и событий, определенных во всех реализованных интерфейсах класса, представляет именуемые свойства в прототипе объекта JavaScript.  Пользователи JavaScript могут получить доступ к любому члену класса напрямую.  Объекты времени выполнения Windows имеют прототип, содержащий все члены из всех реализованных интерфейсов класса.  |
