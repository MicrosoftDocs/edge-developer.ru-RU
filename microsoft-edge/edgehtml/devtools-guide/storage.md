# <a name="storage"></a>Storage

Используйте панель **хранения** для проверки и управления различными локально кэшными данными, в том числе:  

*   [Пары ключей](#local-and-session-storage-managers) и значений для веб-хранилища \(Локальное и сеансное хранилище\)  
*   [Индексные структурированные](#indexeddb-manager) данные DB  
*   [Cookies](#cookies-list) для домена  
*   [Кэш](#cache-manager) \(пары запросов и ответов\) для отладки сотрудника службы  

Расширь любую из этих категорий и нажмите на детскую запись, чтобы открыть вкладку диспетчер ресурсов.  

## <a name="local-and-session-storage-managers"></a>Локальные и сеансные диспетчеры хранения  

Для проверки и управления веб-хранилищем для вашей страницы используйте диспетчер локального хранения и диспетчер хранения сеансов.  

Папки **локального** хранения и хранения сеансов [](./debugger.md#resource-picker) в выборке ресурсов панели хранения отображают список истоков страницы. ****  Выбор одного из этих кадров открывает редактируемую таблицу текущих пар ключей и значений, задаваемых через [Window.localStorage](https://developer.mozilla.org/docs/Web/API/Window/localStorage) или [Window.sessionStorage](https://developer.mozilla.org/docs/Web/API/Window/sessionStorage)соответственно \(и/или задаваемые непосредственно из списка хранения DevTools [](#storage-list)\).  

![Диспетчер хранилища DevTools](./media/storage_web_storage.png)  

Из вкладок локального хранения и хранения сеансов можно:  

*   **Обновление** \. Список хранилища, чтобы увидеть текущий набор пар ключей и значений `Ctrl+F5` для данного [](#cookies-list) домена.  \(Список не обновляется автоматически при обновлении скрипта.\)  
*   **Имитация достижения предельного срока хранения**  для веб-хранилища Microsoft Edge.  У каждого домена и поддомена есть собственная область хранения, однако существует совместное ограничение:  
    *   **Subdomains:** до 5 МБ пространства  
    *   **Домены:** до 10 МБ пространства  
    *   **Итого для всех доменов:** до 50 МБ пространства  

   Хранилище сеансов очищается, как только закрывается последняя вкладка браузера, ссылаясь на ее происхождение.  Локальные записи хранения сохраняются бесконечно до тех пор, пока пользователь не расчистит страницу или вручную:  

   **Параметры**  >  **Очистка данных просмотра**  >  **Файлы cookie и сохраненные данные веб-сайта**  

![Очистка данных просмотра с панели параметров Microsoft Edge](./media/settings_clear_browsing_data.png)  

### <a name="storage-list"></a>Список хранения  

Из таблицы список хранения можно:  

*   **Проверьте и сортировать** пары, нажав на имя `key/value` столбца в таблице.  
*   **Изменение** `Key` `Value` существующей записи и ее изменения, щелкнув в ячейке.  
*   **Удаление** \( \) записи из параметра контекстного меню правой `Del` кнопкой мыши, **Удалить элемент**.  
*   **Добавьте** новую `key/value` пару, щелкнув пустую строку в нижней части таблицы.  

### <a name="shortcuts"></a>Ярлыки

| Действие | Установленное напрямую доверие |  
|:--- |:--- |  
| Обновить | `Ctrl`+`F5` |  
| Удалить элемент | `Del` |  
| Копирование выбранных элементов | `Ctrl`+`C` |  
| Выделить все | `Ctrl`+`A` |  

## <a name="indexeddb-manager"></a>Диспетчер IndexedDB  

Используйте **вкладку IndexedDB** для проверки и управления структурированными данными, хранимыми локально на клиентской машине.  В частности, вы можете проверять/сортировать и обновлять хранилища и индексы объектов, а также удалять отдельные записи с ключевым значением.  

> [!TIP]
> Вы можете использовать нашу [демо-версию](https://developer.microsoft.com/microsoft-edge/testdrive/demos/audiomixer/) аудиомеханик для тест-драйва руководителя *IndexedDB* в Microsoft Edge DevTools.  

Чтобы удалить все данные IndexedDB, хранимые для текущего пользователя в Microsoft Edge, используйте меню Microsoft Edge **Settings:**  

**...** >  **Параметры**  >  **Очистка данных просмотра**  >  **Файлы cookie и сохраненные данные веб-сайта**  

Папка в средстве выборщика ресурсов Отладщика отображает список источников из ресурсов, `IndexedDB` загруженных на [](./debugger.md#resource-picker) страницу.  Все базы данных IndexedDB \(IDB\) будут указаны в начале, а также в их хранилищах объектов.  

![Диспетчер индексации DevTools](./media/storage_indexeddb.png)  

### <a name="indexeddb-toolbar"></a>Панель инструментов IndexedDB  

Из панели инструментов IndexedDB можно:  

*   **Обновление** \. Чтобы увидеть текущие записи `Ctrl` + `F5` в объектном хранилище или индексе базы данных.  Диспетчер IndexedDB не обновляется автоматически при внесении изменений в базу данных.  

### <a name="object-store-entries-list"></a>Список записей в хранилище объектов  

Из таблицы "Объектный магазин" или "Индекс" можно:  

*   **Проверьте и сортировать** пары значений ключа, щелкнув любое имя столбца в таблице.  
*   **Обновление** \( `Ctrl` + `F5` \)  
*   **Удаление элемента** \( `Del` \) для удаления выбранной записи в хранилище объекта или индексе.  Вы также можете сделать это из параметра контекстного меню правой [кнопкой](#context-menu) мыши Удалить **элемент**.  
*   **Скопируйте выбранные** элементы `Ctrl` + \( \) для копирования выбранного элемента `C` в буфер обмена.  Вы также можете сделать это из параметра контекстного меню правой [кнопкой](#context-menu) мыши, **скопируйте выбранный элемент**.  
*   **Выберите все** \( \) для выбора всех `Ctrl` + `A` записей в объектном магазине или индексе.  Вы также можете сделать это из параметра контекстного меню правой [кнопкой](#context-menu) мыши Выберите **все**.  

Столбцы магазина объектов или таблицы Index можно сортировать:  

| Столбец | Описание |  
|:--- |:--- |  
| Раздел | Имя пары ключ-значение \(то же самое, что **Основной**ключ \) при итерации над хранилищем объектов; Имя ключа индекса \(текущий ключ курсора\) при итерации над индексом |  
| Основной ключ | Имя пары ключ-значение \(см. веб-документы **MDN** для получения дополнительных данных на клавишах [IDB](https://developer.mozilla.org/docs/Web/API/IndexedDB_API/Using_IndexedDB#Structuring_the_database)\) |  
| Значение | Значение пары ключ-значение |  

Ознакомьтесь *с веб-docs MDN,* чтобы узнать больше о концепциях [и использовании IndexedDB.](https://developer.mozilla.org/docs/Web/API/IndexedDB_API)

## <a name="cookies-manager"></a>Менеджер cookies  

Используйте диспетчер cookies для проверки и управления файлами cookie для данного домена.  

Папка в средстве выборщика ресурсов Отладщика отображает список источников из ресурсов, `Cookies` загруженных на [](./debugger.md#resource-picker) страницу.  Выбор одного из этих кадров открывает таблицу, представляющую [](https://developer.mozilla.org/docs/Web/HTTP/Cookies) текущие файлы cookie, установленные либо http-загонами, либо по сценарию [с Document.cookie.](https://developer.mozilla.org/docs/Web/API/Document/cookie)  

![Менеджер cookies DevTools](./media/storage_cookies.png)  

На панели инструментов вкладки Cookie можно:  

*   **Обновление** \. `Ctrl` + `F5` Список [cookies,](#cookies-list) чтобы увидеть текущий набор файлов cookie для данного домена.  \(Список не обновляется автоматически.\)  
*   **Удалите все** файлы cookie \( `Ctrl` + `Shift` + `Del` \) \(сеанс и постоянный\) для пути текущей страницы.  
*   **Удалите все файлы cookie сеанса** `Ctrl` + `Del` \( \) для пути текущей страницы.  

Чтобы полностью очистить список cookies, может потребоваться очистить все **файлы cookie** для домена из панели инструментов [панели](./network.md#toolbar) Сети.  

### <a name="cookies-list"></a>Список файлов cookie  

Из таблицы списков Cookie можно:  

*   **Проверьте и сортировать** файлы cookie, щелкнув любое имя столбца в таблице.  
*   **Изменение** `Name` `Value` существующего файла cookie и его редактирования, щелкнув в ячейке.  
*   **Удаление** \( \) cookie из параметра контекстного меню правой `Del` кнопкой [](#context-menu) `Delete cookie` мыши.  
*   **Добавьте** новое cookie сеанса для данного, щелкнув пустую строку `Domain/Path` в нижней части таблицы.  Это работает только для файлов cookie сеанса; постоянные файлы cookie \(с определенными датами истечения\) должны быть установлены традиционными методами.  Значения `Domain` `Path` и значения автоматически заполняются в зависимости от расположения страницы.  

Столбцы списка Cookie можно сортировать:

| Столбец | Описание |  
| :--- | :--- |  
| Name | Имя файла cookie |  
| Значение | Значение файла cookie |  
| Домен | Имя хост-файла cookie \(может быть пустым\) |  
| Путь | ПУТЬ URL-адреса для cookie \(может быть пустым\) |  
| Срок действия истекает | Максимальный срок службы cookie в качестве даты http-даты.  Если нет `Expires` `Max-Age` или установлено, запись считается cookie session.  |  
| ТОЛЬКО HTTP | Указывает, было ли установлено cookie с директивой, указывающее, что он недоступен `HttpOnly` для JavaScript |  
| Защита | Указывает, было ли установлено cookie с директивой, указав, что оно будет отправлено на сервер только из запроса с помощью `Secure` SSL и протокола HTTPS.  |  

Дополнительные сведения о свойствах cookie см. в справке по [веб-docs Set-Cookie](https://developer.mozilla.org/docs/Web/HTTP/Headers/Set-Cookie) **MDN.**
