# Прогрессивное веб-приложение в Microsoft Store

При публикации прогрессивного веб-приложения (PWA) в [Microsoft Store](https://developer.microsoft.com/store)ваша потенциальная аудитория приложения расширяется до всей базы установки Windows 10, вся из почти 700 миллионов активных ежемесячных пользователей! 

PwAs в Microsoft Store имеет ряд преимуществ:

-   **Возможность обнаружения** — приложения в Microsoft Store демонстрируются в различных категориях, коллекциях и фильтрах поиска, что обеспечивает простой просмотр и покупки для потенциальных клиентов вашего приложения. Вы даже можете [улучшить описание в Магазине с](/windows/uwp/publish/app-screenshots-and-images) помощью снимков экрана, главного изображения и видео трейлеров!
-   **Надежность :** клиенты Windows знают, что могут доверять своим покупкам и загрузкам в Microsoft Store, так как они соблюдают строгие стандарты качества [и безопасности Корпорации Майкрософт.](/legal/windows/agreements/store-policies)
-   **Простая** установка — Microsoft Store обеспечивает согласованную и уместную установку во всех приложениях [для Windows 10,](https://www.microsoft.com/store/apps/windows?icid=CNavAppsWindowsApps)что позволяет всем пользователям легко установить PWA независимо от технического уровня удобства.
-   **Бизнес-аналитика.** Информационная панель Центра разработчиков [](/windows/uwp/publish/analytics) [для Windows](/windows/uwp/publish/using-the-windows-dev-center-dashboard) в Microsoft Store предоставляет подробные аналитические сведения обо всем, начиная с того, сколько клиентов Windows было достигнуто в PWA до того, как они используют его и что они должны сказать. Вы также можете найти метрики и данные телеметрии о состоянии приложения, использовании и т. д.
    
Существует два варианта для получения PWA в Microsoft Store:

1.  Вы можете [отправить его вручную](#submitting-your-pwa-manually)или
2.  Если PWA соответствует [определенным критериям,](#criteria-for-automatic-submission)он может автоматически индексироваться веб-обходщиком Bing. [](#automatic-pwa-importing-with-bing) (Вы также можете [](#opting-out-of-automatic-microsoft-store-import) отказаться от автоматической отправки.)
    
Независимо от метода отправки, после присоединения PWA к Microsoft Store вы получите доступ ко всем преимуществам, описанным выше.

## Отправка PWA вручную

Чтобы распространять и продвигать приложение через Microsoft Store, необходимо отправить его в качестве пакета приложения для Windows *(APPX-файл).*  Для веб-приложений с размещенной на сервере, таких как PWAs, этот пакет просто содержит метаданные приложения и значки домашнего экрана (и ни один из фактического кода приложения). Благодаря этому ваше веб-приложение можно установить и запускать на домашнем экране вместе с другими* * приложениями [для Windows 10,](/windows/uwp/get-started/whats-a-uwp) завернув облегченной программы-оболочки (WWAHost.exe), независимо от окна браузера Microsoft Edge.  

> [!IMPORTANT]
> Обдвижок веб-платформы EdgeHTML используется оболочкой приложения WWAHost, что может привести к различиям в совместимости для PWA на основе Microsoft Edge (Chromium).  Перед отправкой приложения в Microsoft Store убедитесь, что приложение прошло полный тестовый проход с помощью Microsoft Edge (EdgeHTML), так как обдвижка EdgeHTML больше не обновляется новыми веб-стандартами.  

Вот как это сделать.

### Предварительные условия

-   **Существующий PWA.** Вот как преобразовать [веб-приложение в PWA,](./get-started.md) если оно еще не является веб-приложением. 
-   **Средство упаковки Windows APPX для PWAs.** Вот как создать и запустить [PWA в Windows](./windows-features.md) с помощью Visual Studio. Вы также можете использовать [построитель PWA](https://www.pwabuilder.com/) для создания пакета Windows.
-   [**Учетная запись разработчика приложений Microsoft Store.**](/windows/uwp/publish/opening-a-developer-account) В зависимости [от](/windows/uwp/publish/account-types-locations-and-fees) типа и расположения выбранной учетной записи взимается разовая плата, а для регистрации требуется действующая [учетная запись Майкрософт.](https://account.microsoft.com/)
    
### Отправка в Магазин *через Visual Studio* 

Выполните следующие [действия, чтобы создать файл](/windows/uwp/packaging/packaging-uwp-apps#create-an-app-package-upload-file) отправки пакета приложения для PWA в Visual Studio. Более общие сведения об этом процессе см. в Visual Studio package a [*UWP*](/windows/uwp/packaging/packaging-uwp-apps) app with Visual Studio.

Эти инструкции также повысят запуск комплекта сертификации приложений [**для Windows,**](https://developer.microsoft.com/windows/develop/app-certification-kit) чтобы протестировать приложение на соответствие требованиям Microsoft Store. Это необязательный, но настоятельно рекомендуемый вариант.

### Отправка в Магазин через *Центр разработчиков для Windows*

Ниже построитель *PWA* используется для создания пакета приложения для отправки в Центр разработчиков для Windows.

1.  Создание приложения для Windows 10. Вот как запустить [PWA](./windows-features.md)в качестве приложения для Windows с Visual Studio. Вы также можете использовать [построитель PWA](https://www.pwabuilder.com/) для создания приложения для Windows 10.
2.  Зарезервировать имя приложения для Microsoft Store, войдя в информационную панель Центра разработчиков [для Windows](https://developer.microsoft.com/dashboard/windows/overview) с информацией об учетной записи и выйдя на его создание, зарезервировать [имя.](/windows/uwp/publish/create-your-app-by-reserving-a-name) Каждое зарезервированное имя должно быть уникальным в Microsoft Store.
3.  При отправке пакетов приложения значения *DisplayName,* *Name,* *Publisher*и *PublisherDisplayName* в файле *.appxmanifest* должны совпадать со значениями, присвоенным приложению на информационной панели Центра разработчиков для Windows после его засчета. 
    
    Войдите на информационную [панель Центра разработчиков для Windows](https://developer.microsoft.com/dashboard/windows/overview)и найдите значения удостоверения приложения в **удостоверении**приложения управления  >  **приложениями:**
    
    ![Информационная панель Центра разработчиков для Windows, параметры удостоверения приложения](./media/dashboard-app-identity.png)
    
    Затем найдите файл и замените следующие значения на те, которые назначены на информационной панели `appxmanifest.xml` Центра разработчиков для Windows:
    
    -   **<Identity Name="** *Package/Identity/Name*
    -   **<Identity Publisher="** *Package/Identity/Publisher*
    -   **<DisplayName** **>** *Имя, зарезервированное для вашего приложения* 
    -   **<PublisherDisplayName** **>** *Package/Properties/PublisherDisplayName***</PublisherDisplayName>**
        
4.  Теперь все ресурсы PWA можно скомпилировать в один файл, который можно `.appx` отправить в Microsoft Store. В командной подсказке перейдите в каталог веб-манифеста и создайте пакет Windows 10 (указанный ниже, если вы не можете вести журнал отлаживки).
    
    ```shell
    pwabuilder package -p windows10 -l debug
    ```  
    
    AppX-файл будет создан в этом расположении: `PWA\Store packages\windows10\package\windows.appx` .
    
5.  Перед отправкой приложения в Microsoft Store для подчиненного ему стоит проверить его на соответствие политикам Microsoft Store. Это можно сделать, скачав комплект сертификации приложений [**для Windows,**](https://developer.microsoft.com/windows/develop/app-certification-kit)запустив его, а затем выбрав *APPX-файл* приложения для тестирования. После завершения всех тестов вы получите отчет о областях, которые необходимо решить.
6.  Загрузите пакет и завершите отправку, снова войдя в **** информационную панель Центра разработчиков [для Windows](https://developer.microsoft.com/dashboard/windows/overview) и расширив панель "Подмывка  >  **отправки 1".** Следуйте контрольным спискам, чтобы заполнить [необходимые](/windows/uwp/publish/app-submissions) сведения о списке в магазине [и отправить пакет приложения.](/windows/uwp/publish/upload-app-packages)
    
Подробнее о всех доступных разработчикам приложений Microsoft Store функций и служб см. в публикации приложений [*для Windows*](https://developer.microsoft.com/store/publish-apps) в [Центре разработки для Windows.](https://developer.microsoft.com/windows)

## Автоматический импорт PWA с помощью Bing

Так же как манифест [](https://developer.mozilla.org/docs/Web/Manifest) веб-приложения PWA является сигналом для поддержки платформ, что ваше приложение может работать в автономном режиме и готово к тому, чтобы его можно было представить как полностью интегрированное приложение, это также является подсказкой для веб-обходщика Bing о том, что PWA следует рассматривать для автоматического включения в Microsoft Store. 

Если PWA соответствует требованиям ниже, служба индексации Bing автоматически упаковит PWA в формат [*APPX,*](#submitting-your-pwa-manually) необходимый для установки в Windows 10, и соберет страницу продукта магазина вашего приложения на основе метаданных в манифесте веб-приложения. После утверждения PWA вы сможете дополнительно настроить страницу Магазина и получить доступ к аналитике пользователей и другим средствам управления приложениями через информационную панель Центра разработчиков для Windows.

### Критерии автоматической отправки

Для автоматической упаковки и перечисления в Microsoft Store вашему PWA потребуется:

-   Непустый файл манифеста веб-приложения, как минимум:
    
    -   Имя
    -   Описание
    -   Значок приложения размером не менее 512 x 512 пикселей
        
-   Уникальная основная логика (не минимально измененный код из шаблона [приложения)](https://en.wikipedia.org/wiki/Boilerplate_code)
-   Для безопасного подключения (HTTPS)
-   Скрипты рабочих служб
-   Чтобы не нарушать законы или политики [Microsoft Store.](/legal/windows/agreements/store-policies)
    
Мы не будем использовать все приложения, удовлетворяющие этим критериям, но включим их в свои вопросы для кандидатов по мере постепенного расширения нашей программы.

### Отказ от автоматического импорта в Microsoft Store

PWA может отказаться от автоматического импорта в Microsoft Store, обслуживая файл, содержащий `robots.txt` следующие данные:

```text
User-agent: bingbot
Disallow: /manifest.json
```  

Это направляет веб-обходщик Bing (Bingbot) игнорировать файл веб-манифеста в целях индексации PWA. Это не повлияет на ранжирование поиска сайта в обычном процессе веб-индексирования [Bing.](https://www.bing.com/webmaster/help/help-center-661b2d18)
